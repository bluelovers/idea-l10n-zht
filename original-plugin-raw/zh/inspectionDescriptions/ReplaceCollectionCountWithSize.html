报告对 Collection<T>.count() 的调用。

此函数调用可以替换为 .size。

.size 形式可以确保运算是 O(1) 并且不会分配额外的对象，而 count() 可能与 Iterable<T>.count() 混淆，后者是 O(n) 和分配。

示例：

  fun foo() {
      var list = listOf(1,2,3)
      list.count() // 可替换的 'count()'
  }

在应用快速修复后：

  fun foo() {
      var list = listOf(1,2,3)
      list.size
  }
