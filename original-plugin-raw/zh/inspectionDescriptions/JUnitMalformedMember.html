报告格式错误且可能无法被 JUnit 框架识别的 JUnit 成员声明。 本次检查报告的问题如下：
  • 使用 @RegisterExtension 注解的字段，它们的类型错误或者在需要时未声明为 static。
  • 使用 @Nested 注解的 static 内部类。
  • 在没有源的情况下定义的形参化测试。
  • 具有未知、非 static 或无实参目标的 MethodSource 的形参化测试。
  • 具有无法转换为方法形参的 ValueSource 或 EnumSource 的形参化测试。
  • 同时使用 `@Test`、`@ParameterizedTest` 或 `@RepeatedTest` 注解的测试。
  • setup() 或 tearDown()
    • 非 public 方法，其返回类型不是 void 或接受实参。
  • 由 @BeforeClass、@AfterClass、@BeforeAll 或 @AfterAll 注解的非 public，非 static 方法，其返回类型不是 void 或没有有效的形参列表。
  • 由 @Before、@After、@BeforeEach 或 AfterEach 注解的非 public 方法，其返回类型不是 void 或接受实参。
  • 在 @BeforeAll 或 @AfterAll 方法中注入的 RepetitionInfo。
  • 由 @Test 注解测试使用的，在 @BeforeEach 或 @AfterEach方法中注入的 RepetitionInfo。
  • 由 @DataPoint 或 @DataPoints 注解的非 public 或非 static 字段和方法。
  • 由 @Rule 注解的非 public 或非 TestRule 或 MethodRule 的子类型的字段和方法。
  • 由 @ClassRule 注解的非 public，非 static 或非 TestRule 的子类型的字段和方法。
  • TestCase 内带有 test 前缀的非 public 方法，其返回类型不是 void，接受实参或为 static。
  • 由 @Test 注解的非 public 方法，其返回类型不是 void，接受实参或为 static。

格式错误的 @Before 方法示例 (Java)：

@Before private int foo(int arg) { ... }

在应用快速修复后：

@Before public void foo() { ... }

缺少方法源代码示例 (Kotlin)：


  class Example {
    @MethodSource("parameters")
    @ParameterizedTest
    fun foo(param: String) { ... }
  }

在应用快速修复后：


  class Example {
    @MethodSource("parameters")
    @ParameterizedTest
    fun foo(param: String) { ... }

    companion object {
      @JvmStatic
      fun parameters(): Stream<Arguments> {
        TODO("Not yet implemented")
      }
    }
  }

使用检查选项可以指定注解。 不会在测试方法中报告任何使用其中一种注解进行注解的形参。
