报告从 1.7 版开始会导致编译错误的非详尽 when 语句。

动机类型：

  • 需要劝阻/阻止的有问题/无意义的用法模式（例如反直觉的行为）
    • 代码容易出错
  • 设计中的不一致（在不同的上下文中存在不同的处理方式）

影响类型：

  • 编译。 一些过去可以编译的代码已无法编译
    • 在某些情况下，此类代码可以无异常地工作
      • 某些此类代码可以在没有任何警告的情况下编译

更多详细信息： KT-47709：默认情况下使带有 enum、sealed 和 Boolean 对象的 when 语句详尽

快速修复会添加缺少的 else -> {} 分支。

示例：


  sealed class Base {
      class A : Base()
      class B : Base()
  }

  fun test(base: Base) {
      when (base) {
          is Base.A -> ""
      }
  }

在应用快速修复后：


  sealed class Base {
      class A : Base()
      class B : Base()
  }

  fun test(base: Base) {
      when (base) {
          is Base.A -> ""
          else -> {}
      }
  }

仅当项目或模块的 Kotlin 语言级别为 1.6 或更高时，此检查才会报告。
