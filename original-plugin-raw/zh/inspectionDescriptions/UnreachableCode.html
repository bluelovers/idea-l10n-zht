根据数据流分析报告从未到达的代码。 可能是前一个始终为 true 或始终为 false 的条件、无法到达的循环主体或 catch 部分的结果。 通常（尽管并非总是）无法到达的代码是先前警告的结果，因此请检查形成“为 null 性和数据流问题”、“常量值”或“空容器上的冗余操作”的检查警告，以更好地了解原因。

示例：


  void finishApplication() {
    System.exit(0);
    System.out.println("Application is terminated"); // 不可到达的代码
  }

请注意，此检查依赖于方法契约推断。 特别是，如果您调用总是引发异常的 static 或 final 方法，那么将推断出“总是失败”契约，并且方法调用之后的代码将被视为不可到达。 示例： 


  void run() {
    performAction();
    System.out.println("Action is performed"); // 不可到达的代码
  }
  
  static void performAction() {
    throw new AssertionError();
  }

如果使用任何类型的代码后处理，这可能会导致误报，例如，如果注解处理器稍后用有用的内容替换了方法体。 To avoid false-positive warnings, one may suppress the automatic contract inference with explicit @org.jetbrains.annotations.Contract annotation from org.jetbrains:annotations package: 


  void run() {
    performAction();
    System.out.println("Action is performed"); // 不再有警告
  }

  @Contract("-> _") // 实现将被替换
  static void performAction() {
    throw new AssertionError();
  }
