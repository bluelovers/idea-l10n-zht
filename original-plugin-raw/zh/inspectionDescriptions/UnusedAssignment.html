报告赋值后因从未使用过其值而导致变量冗余的情况。

如果不使用该变量，最好将其移除，以缩短代码并避免冗余分配。

报告的情况如下:

  • 赋值后从未读取该变量。
  • 在读取下一个变量之前，该值总是被另一个赋值覆盖。
  • 变量初始值设定项冗余（出于以上两种原因之一）。

配置检查：

使用报告冗余初始化器选项，以报告冗余初始化器: 


  int getI() {
    int i = 0; // 冗余初始化
    i = 2;
    return i;
  }

使用在可以替换为 (i + 1) 时报告 ++i 选项，报告 ++i 表达式可以替换为 i + 1 的情况: 


  int preInc(int i) {
    int res = i;
    return ++res;
  }

使用在后续未使用变更后的值时报告 i++ 选项，报告后续不使用 i++ 表达式的结果的情况: 


  int postInc(int i) {
    int res = i;
    return res++;
  }
