报告不必要地声明为 final 的局部变量或形参。

出于简洁的原因，某些编码标准不赞成声明为 final 的变量。

示例：


  class Foo {
    Foo(Object o) {}

    void bar(final Object o) {
      new Foo(o);
    }
  }

在应用快速修复后：


  class Foo {
    Foo(Object o) {}

    void bar(Object o) {
      new Foo(o);
    }
  }

使用检查选项可以切换以下各项的报告：

  • 局部变量
  • 形参（包括 catch 块的形参和增强型 for 语句）

此外，您可以将检查配置为仅报告 abstract 或接口方法的 final 形参，这可能被认为是多余的，因为此类标记不会影响这些方法的实现。
