如果不想在每次修改 go.mod 後執行 go list ，請按 &shortcut:ShowSettings; 開啟設定，並導覽至 Build, Execution, Deployment | Build Tools 。清除 Reload project after changes in the build scripts 複選框。

預設情況下，go.mod 檔案發生任何變化後 GoLand 都會重新載入專案。但是如果選擇 External changes 選項，則在 IDE 中編輯檔案時 go list 不會自動執行。這種情況下，您將看到“載入 Go 模組變更”圖標。完成編輯並點擊該圖標以載入變更。
