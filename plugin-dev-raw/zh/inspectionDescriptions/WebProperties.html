報告用於將 Web 模組部署到伺服器的檔案描述子中的以下問題：
  • 對不可實例化類別的參照
  • 對不擴展所需類別的類別的參照
  • 對範圍不適當的類別的參照
  • 空標記和特性值
  • 與所需模式不符合的標記和特性值（例如，Java 關鍵字）
  • 不包含必需子標記或特性的標記
  • 定義具有重複名稱的物件的標記

範例：



<web-app>
  <filter-mapping>
    <filter-name>CustomFilter</filter-name> // 錯誤：應定義 servlet-name 或 url-pattern 子標記
  </filter-mapping>
  <servlet-mapping>
    <servlet-name>CustomServletName</servlet-name>
    <url-pattern>/ baseUrl</url-pattern> // 錯誤：URI 模式不得包含空格
  </servlet-mapping>
</web-app>
