報告任何可以移動到更小範圍的變數宣告。

這種檢查對於方法開頭的 Pascal 樣式宣告特別有用。 此外，範圍太廣的變數在重構後也通常會被留下。

範例：


  StringBuilder sb = new StringBuilder();
  System.out.println();
  sb.append(1);

在套用快速修復後：


  System.out.println();
  StringBuilder sb = new StringBuilder();
  sb.append(1);

設定檢查：

  • 使用僅報告可移入內部塊的變數選項可只報告那些可以移動到更深程式碼塊中的變數。 例如，啟用該選項時，將不建議對上面的 sb 變數進行移動。 但是，建議將其用於以下程式碼： 
    
  StringBuilder sb = new StringBuilder(a);
  if (flag) {
    sb.append(1);
  }
  • 使用報告使用新表達式作為初始設定式的變數(可能不安全) 選項可以報告使用新表達式初始化的變數。 這使得當構造函數具有非本地副作用時，檢查可能會不安全。 例如，啟用該選項時，將建議對 foo 變數進行移動： 
    
  class Foo {
    static List<Foo> fooList = new ArrayList<>();
    String bar;

    Foo(String bar) {
        this.bar = bar;
        fooList.add(this);
    }

    public static void main(String[] args) {
      // 雖不安全，但也可以移動
      Foo foo = new Foo("bar");
      System.out.println(fooList.size());
      System.out.println(foo.bar);
    }
  }
