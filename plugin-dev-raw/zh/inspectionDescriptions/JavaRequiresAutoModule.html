報告 requires 指令中自動模組的用法。

自動模組不可靠，因為它可能依賴於類路徑中的類型，而且當它轉換為顯式模組時，它的名稱和匯出的軟體套件可能會改變。

對應於 -Xlint:requires-automatic 和 -Xlint:requires-transitive-automatic Javac 選項。 第一個選項增加了對何時使用自動模組的了解。 第二個選項則是警示模組的作者，建立自動模組的隱含可讀性將使該模組的使用者面臨風險。

範例：


  //module-info.java
  module org.printer {
    requires transitive drivers.corp.org; // 在 'drivers.corp.org' 為自動模組時報告
  }

使用僅高亮顯示傳遞依賴選項，可僅對傳遞依賴發出警告。
