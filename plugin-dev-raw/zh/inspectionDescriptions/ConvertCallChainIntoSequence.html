報告應轉換為 Sequence 的 Collection 上的調用鏈。

每個 Collection 轉換函數（例如 map() 或 filter()）都會在後台創建一個新的 Collection（通常是 List 或 Set）。 多次後續調用以及 Collection 中存在大量條目的情況下，內存流量可能會很大。 這種情況下，最好使用 Sequence。

範例：


  class Entity(val key: String, val value: String)

  fun getValues(lines: List<String>) = lines
      .filter { it.isNotEmpty() }
      .map { it.split(',', limit = 2) }
      .filter { it.size == 2 }
      .map { Entity(it[0], it[1]) }

快速修復可將調用鏈包裝到 asSequence() 和 toList() 中：


  class Entity(val key: String, val value: String)

  fun getValues(lines: List<String>) = lines
      .asSequence()
      .filter { it.isNotEmpty() }
      .map { it.split(',', limit = 2) }
      .filter { it.size == 2 }
      .map { Entity(it[0], it[1]) }
      .toList()
