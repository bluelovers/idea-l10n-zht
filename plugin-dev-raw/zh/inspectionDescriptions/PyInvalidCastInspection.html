報告呼叫 `typing.cast` 時，源類型的任何可能值都無法分配給目標類型。 我們可以將這稱為“非重疊”類型。

這通常表明有錯誤。 如果轉換是故意的，首先將表達式轉換為公共父類型，以明確表達意圖。

範例：


from typing import cast

# 非重疊類型 — 可能是一個錯誤
cast(int, "a")          # 'str' -> 'int'
cast(list[int], ["a"])  # 'list[str]' -> 'list[int]'

# 推薦的顯式逃避方法是使用 “雙重轉換”
cast(int, cast(object, "a"))  # ok

# 合法的重疊案例
cast(int, object())    # 有效的向下轉換
cast(object, 1)        # 有效的向上轉換

# 雖然以下是無效轉換，因為 list 是不可變的。 當前不支持此檢查
int_list = [1, 2, 3]
cast(list[object], int_list)

檢查依賴於靜態類型資訊; 當類型未知時，不會報告警告。 尚未考慮泛型類型的變化。
