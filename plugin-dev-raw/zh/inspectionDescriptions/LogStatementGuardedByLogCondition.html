報告帶有未被保護條件包圍的非常數實參的日誌記錄呼叫。 對日誌記錄呼叫的實參的評估可能開銷巨大。 當日誌記錄語句使用的級別停用日誌記錄時，使用臨界子句包圍日誌記錄呼叫可避免這種開銷。 這對於最不嚴重的日誌記錄呼叫級別（追蹤、偵錯、最精細）特別有用，因為這些呼叫在生產環境中最常被停用。

範例：


  public class Principal {
    void bad(Object object) {
      if (true) {
        LOG.debug("log log log " + expensiveCalculation(object));
      }
      LOG.debug("some more logging " + expensiveCalculation(1));
    }

    void good(Object) {
      if (LOG.isDebug()) {
        LOG.debug("value: " + expensiveCalculation(object));
      }
    }
  }

組態檢查：

  • 使用記錄器類別名欄位可以指定使用的記錄器類別名。

  • 使用該表可以指定此檢查應警告的日誌記錄方法，以及相應的日誌條件文本。

  • 使用標記所有不受保護的日誌呼叫選項可讓檢查標記所有不受保護的日誌呼叫，而不僅僅是那些具有非常數實參的呼叫。
