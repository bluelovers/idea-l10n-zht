報告類型參數被忽略的泛型類別。 這種對泛型類型的原始使用在 Java 中雖然有效，但它違背了類型參數的用途，並且可能掩蓋錯誤。 此檢查反映出 javac 的 rawtypes 警告。

範例：


//警告：參數化類別 'List' 的原始使用
List list = new ArrayList<String>();
//建立了字串列表，但也接受整數
list.add(1);


//無警告，因為在陣列建立期間無法提供類型實參
IntFunction<List<?>[]> fun = List[]::new;

組態檢查：

  • 使用忽略新物件的建構選項，可忽略物件建構中使用的原始類型。
  • 使用忽略類型轉換選項，可忽略類型轉換中使用的原始類型。
  • 使用忽略類型參數無法編譯的位置選項，可忽略類型參數編譯失敗的情況（例如建立陣列或覆寫庫方法的情況）。
  • 使用忽略覆寫方法的參數類型選項，可忽略覆寫方法的參數中使用的類型參數。
  • 使用在自動快速修復不可用時忽略選項，可忽略快速修復不可用的情況。
僅當專案或模組的語言級別為 5 或更高時，此檢查才會報告。
