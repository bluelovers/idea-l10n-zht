報告未安全關閉的 JDBC 資源。 此檢查報告的 JDBC 資源包括 java.sql.Connection、java.sql.Statement、java.sql.PreparedStatement、java.sql.CallableStatement 和 java.sql.ResultSet。

預設情況下，該檢查假定資源可以由名稱中帶有 'close' 或 'cleanup' 的任何方法關閉。

範例：


  ResultSet findAllElements(Connection connection) throws SQLException {
    PreparedStatement statement = connection.prepareStatement("SELECT * FROM TABLE");//語句未關閉
    statement.execute();
    return statement.getResultSet();
  }

使用以下選項組態檢查：

  • 是否允許在 try 塊內開啟 JDBC 資源。 這種樣式不太合意，因為它比在 try 塊前面開啟資源更冗長。
  • 資源能否通過任何將資源作為實參傳遞的方法呼叫來關閉。
