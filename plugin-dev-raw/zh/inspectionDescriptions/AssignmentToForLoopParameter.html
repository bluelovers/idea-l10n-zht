報告對循環體內 for 迴圈參數的賦值或修改。

雖然偶爾是有意為之，但此結構可能令人困惑，通常是由拼寫錯誤或使用錯誤變數引起。

該快速修復會新增新變數的宣告。

範例：


  for (String s : list) {
    // 警告：s 在迴圈內被更改
    s = s.trim();
    System.out.println("String: " + s);
  }

在套用快速修復後：


  for (String s : list) {
    String trimmed = s.trim();
    System.out.println("String: " + trimmed);
  }

不報告沒有 update 語句的基本 for 迴圈中的賦值。 在此類別情況下，賦值很可能是預期的，不能輕易移動到 for 迴圈的更新部分。

範例：


  for (int i = 0; i < list.size(); ) {
    if (element.equals(list.get(i))) {
      list.remove(i);
    } else {
      // 因為沒有 update 語句
      // 所以不報告 for 迴圈參數的修改
      i++;
    }
  }

使用檢查增強型 'for' 迴圈參數選項可指定是否還應報告增強的 for 迴圈參數的修改。
