報告將沒有 __ toString 方法實作的物件轉換為 string 的嘗試。

在 PHP 7.4 之前，這將導致一個致命的 E_RECOVERABLE_ERROR 級錯誤。 在 PHP 7.4 中，會拋出一個致命的不可恢復的 ERROR。 從 PHP 8.0 開始，返回值遵循標準 PHP 類型語意，如果可能且嚴格類型模式被停用，就會被強制轉換為字串。

如果啟用了檢查每種表達式類型是否存在 __toString 選項，則該檢查將會檢查表達式的所有可能類型，如果其中至少一種類型不包含 __toString 方法實作，則進行報告。

有關詳細資訊，請參閱 __toString (php.net)。
