報告雙大括號初始化。

雙大括號初始化在非 static 上下文中使用時可能會導致記憶體泄漏，因為它創建了一個將引用周圍物件的匿名類。

與一般初始化相比，雙大括號初始化的性能更差，因為它需要載入一個額外的類。

如果 equals() 方法不接受子類作為參數，還可能導致 equals() 比較失敗。

此外，在 Java 9 以前的版本中，雙大括號初始化不能與 Diamond 運算符結合使用，因為它與匿名類不相容。

範例：


  List<Integer> list = new ArrayList<>() {{
    add(1);
    add(2);
  }};

在套用快速修復後：


  List<Integer> list = new ArrayList<>();
  list.add(1);
  list.add(2);
