報告空的 finally 塊。

空的 finally 塊通常表明存在編碼錯誤。 在程式碼重構後它們可能保留下來，可以安全移除。

此檢查不報告在 JSP 檔案中發現的空 finally 塊。

範例：


  try {
    Files.readString(Paths.get("in.txt"));
  } catch (IOException e) {
    throw new RuntimeException(e);
  } finally {

  }

在套用快速修復後： 


  try {
    Files.readString(Paths.get("in.txt"));
  } catch (IOException e) {
    throw new RuntimeException(e);
  }
