報告未在 try-with-resources 語句中使用的 AutoCloseable 實例，也稱 自動資源管理。

這意味著還報告在 try-with-resources 可用前使用的“在 try 前面/中開啟資源，在 finally 中關閉資源”的樣式。 在 Java 7 及更高版本中開發時，此檢查意在取代所有已開啟但未安全關閉的檢查。

範例：


  private static void foo() throws IOException {
    InputStream profile = Thread.currentThread().getContextClassLoader().getResourceAsStream("/someFile");
    System.out.println(profile.read());
  }

使用以下選項組態檢查：

  • 列出不需關閉並且此檢查應當忽略的 AutoCloseable 的子類別。
    注意：即便列出將忽略 java.util.stream.Stream，該檢查仍將報告從 java.nio.file.Files 方法 lines()、walk()、list() 以及 find() 返回的流。 這些流包含需要關閉的關聯 I/O 資源。
  • 列出在呼叫時應被忽略的返回 AutoCloseable 的方法。
  • 如果為方法呼叫的結果時是否忽略 AutoCloseable。 啟用此選項時，工廠方法的結果也將被忽略。
  • 當 AutoCloseable 實例作為方法呼叫實參傳遞時，該檢查是否應報告。 如果啟用此選項，該檢查將假定所呼叫方法中的資源已關閉。 不會忽略名稱中含有 'close' 的 finally 塊和 AutoCloseable 實參中的方法呼叫。
  • 是否忽略對資源類別建構函式的方法參照。
  • 是否忽略返回資源且名稱以 'get' 開頭的方法。 這可能會減少誤報，因為大多數 getter 不轉移資源的所有權，並且其呼叫點不負責關閉資源。
