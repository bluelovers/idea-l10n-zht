報告對 java.sql.Statement.execute() 或其任何變體的調用，這些變體會將動態構造的字串作為要執行的查詢。

構造的 SQL 語句是安全漏洞的常見來源。 預設情況下，此檢查會忽略編譯時常量。

範例：


  ResultSet execute(Statement statement, String name) throws SQLException {
    return statement.executeQuery("select * from " + name); // 報告警告
  }

使用檢查選項可以將任何 static final 欄位視為常數。 請小心，因為啟用該選項時，像下面這樣的字串將被忽略： 


  private static final String SQL = "SELECT * FROM user WHERE name='" + getUserInput() + "'";
