報告參數比相應的 try 塊拋出的異常更通用的 catch 塊。

範例：


  try  {
    File file = new File(pathToFile);
    return file.getAbsolutePath();
  } catch (Exception ex) { // warning: 'catch' of 'Exception' is too broad, masking exceptions 'RuntimeException'
    return defaultFilePath;
  }

在套用快速修復後：


  try  {
    File file = new File(pathToFile);
    return file.getAbsolutePath();
  } catch (RuntimeException ex) {
    return defaultFilePath;
  }

設定檢查：

  • 使用僅對 RuntimeException、異常、錯誤或 Throwable 發出警告選項可使此檢查僅對最常見的異常發出警告。
  • 使用忽略隱藏其他異常但自身被拋出的異常選項可忽略任何隱藏其他異常但仍可能被拋出的異常，因此在技術上不會過於寬泛。
