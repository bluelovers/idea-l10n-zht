此檢查可以檢測潛在的 nil 解參照，並考慮所有函式呼叫。 該分析是跨過程的，並且會考慮整個專案中的呼叫，包括不同軟體套件之間的呼叫。

範例：


package a

func CreateUser(name string) *User {
  if isValid(name) {
    return &User{Name: name}
  }
  return nil
}


package main

func main() {
	user := CreateUser("NilDereference")
	print(user.Name) // 潛在的 nil 解參照
}

在提供的範例中，user 是指向 User 結構體的指針，但它可能為 nil，因為 CreateUser 可能會返回nil。 嘗試解參照 user 以存取 Name 而首先不檢查 user 是否為 nil 會導致執行時錯誤。
