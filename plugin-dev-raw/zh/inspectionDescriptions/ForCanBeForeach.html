報告迭代集合或陣列並且可以自動取代為增強型 for 迴圈（foreach 迭代語法）的 for 迴圈。

範例：


  for (Iterator<String> iterator = list.iterator(); iterator.hasNext(); ) {
    String item = iterator.next();
    System.out.println(item);
  }

在套用快速修復後：


  for (String item : list) {
    System.out.println(item);
  }

使用報告索引的 'java.util.List' 迴圈選項可尋找涉及 list.get(index) 呼叫的迴圈。 這些迴圈通常可以取代為增強型 for 迴圈，除非它們修改了程序中的基礎列表，例如通過呼叫 list.remove(index) 進行修改。 如果是後面這種情況，增強型 for 迴圈可能拋出 ConcurrentModificationException。 在某些情況下，list.get(index) 迴圈的動作速度也可能更快一些。

使用不報告無類型集合上的迭代選項可忽略沒有類型參數的集合。 這將防止建立 java.lang.Object 類型的增強型 for 迴圈變數以及在使用迴圈變數的位置插入轉換。

僅當專案或模組的語言級別為 5 或更高時，此檢查才會報告。
