報告從嵌套類到外部類的非常數 private 成員的引用。 對於此類引用，javac 將生成 package-private 合成訪問器方法，這可能會危及安全性，因為看似為 private 的成員實際上可以從整個軟體套件中訪問。

嵌套類及其外部類會被編譯為單獨的類檔案。 Java 虛擬機通常會禁止從一個類訪問另一個類的 private 欄位和方法。 為了實現從嵌套類訪問外部類的 private 成員，javac 創建了一個 package-private 合成訪問器方法。

通過將 private 成員設為 package-private，實際的可訪問性變成了顯式。 這也會節省少量記憶體，可能會在資源受限的環境中提高性能。

僅當專案或模組的語言級別為 10 或更低時，此檢查才會報告。 在 Java 11 和更高版本中，由於基於嵌套的訪問控制(JEP 181)，不再生成訪問器方法。

範例：


  class Outer {
    private void x() {}

    class Inner {
      void y() {
        x();
      }
    }
  }

在套用快速修復後：


  class Outer {
    void x() {}

    class Inner {
      void y() {
        x();
      }
    }
  }
