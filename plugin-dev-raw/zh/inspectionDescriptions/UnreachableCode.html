根據數據流分析報告從未到達的程式碼。 可能是前一個始終為 true 或始終為 false 的條件、無法到達的迴圈主體或 catch 部分的結果。 通常（儘管並非總是）無法到達的程式碼是先前警告的結果，因此請檢查形成“為 null 性和數據流問題”、“常量值”或“空容器上的冗餘動作”的檢查警告，以更好地了解原因。

範例：


  void finishApplication() {
    System.exit(0);
    System.out.println("Application is terminated"); // 不可到達的程式碼
  }

請注意，此檢查依賴於方法契約推斷。 特別是，如果您呼叫總是引發異常的 static 或 final 方法，那麼將推斷出“總是失敗”契約，並且方法呼叫之後的程式碼將被視為不可到達。 範例： 


  void run() {
    performAction();
    System.out.println("Action is performed"); // 不可到達的程式碼
  }
  
  static void performAction() {
    throw new AssertionError();
  }

如果使用任何類型的程式碼後處理，這可能會導致誤報，例如，如果註解處理器稍後用有用的內容取代了方法體。 To avoid false-positive warnings, one may suppress the automatic contract inference with explicit @org.jetbrains.annotations.Contract annotation from org.jetbrains:annotations package: 


  void run() {
    performAction();
    System.out.println("Action is performed"); // 不再有警告
  }

  @Contract("-> _") // 實作將被取代
  static void performAction() {
    throw new AssertionError();
  }
