報告不符合標準 Java 語法的格式字串。

預設情況下，如果編譯時常量被用作 java.util.Formatter、java.lang.String、java.io.PrintWriter 或 java.io.PrintStream 上相應方法的實參，則該檢查會將其視為格式字串。

範例：


  String.format("x = %d, y = %d", 42);

使用檢查設定可以將其他類別和方法標記為與字串格式相關。

作為替代方案，您可以使用 org.intellij.lang.annotations.PrintFormat 註解 來標記格式字串方法參數。 在這種情況下，格式實參參數必須緊跟在格式字串之後，並且必須是最後一個方法參數。 範例：


  void myFormatMethod(int mode, @PrintFormat String formatString, Object... args) {...}

以這種方式註解的方法也將被此檢查識別。
