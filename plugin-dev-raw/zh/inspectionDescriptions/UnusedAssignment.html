報告賦值後因從未使用過其值而導致變數冗餘的情況。

如果不使用該變數，最好將其移除，以縮短程式碼並避免冗餘分配。

報告的情況如下:

  • 賦值後從未讀取該變數。
  • 在讀取下一個變數之前，該值總是被另一個賦值覆蓋。
  • 變數初始設定式冗餘（出於以上兩種原因之一）。

設定檢查：

使用報告冗餘初始化器選項，以報告冗餘初始化器: 


  int getI() {
    int i = 0; // 冗餘初始化
    i = 2;
    return i;
  }

使用在可以取代為 (i + 1) 時報告 ++i 選項，報告 ++i 表達式可以取代為 i + 1 的情況: 


  int preInc(int i) {
    int res = i;
    return ++res;
  }

使用在後續未使用變更後的值時報告 i++ 選項，報告後續不使用 i++ 表達式的結果的情況: 


  int postInc(int i) {
    int res = i;
    return res++;
  }
