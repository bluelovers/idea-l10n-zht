最常見的情況是在類別中宣告一個欄位，然後在方法中使用同名的局部值或變數：


class Foo(foo: Int) {
  def test(): Int = {
    val foo: Int = 1 // 高亮顯示
    foo
  }
}

這很容易理解，但如果在另一個檔案的超類別或特徵中宣告該欄位，則會變得更加麻煩，這樣做的話，該欄位可能會更不明顯，且更容易出錯：


// TraitA.scala
trait TraitA {
  def n: Int = 1
}

// ObjectA.scala
object ObjectA extends TraitA {
  def foo(): Unit = {
    val n: Int = 0 // 高亮顯示
  }
}

在極端情況下，類別定義變數，而子類定義同名的值。 子類值不能覆蓋超類別變數，只能隱藏它。 這有時會導致非普通錯誤。 Scala 編譯器採用選項 -Xlint:private-shadow，會在這種情況下發出警告。

“選項”部分中的組合框允許控制如何高亮顯示此極端情況。 您可以選擇始終高亮顯示，以及高亮顯示隱藏類別或特徵欄位的局部欄位，您可以選擇永不高亮顯示，並保留其他高亮顯示，您可以選擇選中編譯器選項並讓檢查做出相應行為，或者您可以選擇僅高亮顯示極端情況（如果存在編譯器選項），而不高亮顯示其他隱藏情況。


class C(var c: Int)

// `c` shadows `var c` from the `C` class
class D(c: Int) extends C(c) {
  def f: Int = c
}
