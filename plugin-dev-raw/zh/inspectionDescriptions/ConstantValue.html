報告始終產生相同結果的表達式和條件，如 true、false、null 或 0。 此類別表達式可被取代為相應的常量值，或者可能表明程式碼中存在 bug。

範例：

  // 始終為 true
  // 原因：使用了 ||，而非 &&
  if (x > 0 || x < 10) {}

  System.out.println(str.trim());
  // 始終為 false
  // 原因：在 null 檢查之前變數被解參照
  if (str == null) {}

檢查行為可能由註解控制，例如為 null 性註解、@Contract 註解、@Range 註解等。

組態檢查：

  • 使用不報告條件靜態證明始終為 true 的斷言選項可避免報告靜態證明始終為 true 的斷言。 這包括 if (alwaysFalseCondition) throw new IllegalArgumentException(); 等條件。
  • 使用忽略 assert 語句選項可控制檢查如何處理 assert 語句。 預設情況下，斷言會被假定為已執行（-ea 模式）。 當該選項啟用時，斷言將被忽略（-da 模式）。
  • 使用當常數存儲在變數中時發出警告選項可以在使用已知其值為常數的變數時顯示警告。

在 IntelliJ IDEA 2022.3 版本之前，此檢查為常數條件和異常檢查的一部分。 此檢查已被拆分為兩個檢查：常量值與為 null 性和數據流問題。
