報告實作 Cloneable 接口但不覆寫 clone() 方法的類別。

這種類使用 clone() 的預設實作，它不是 public，而是 protected，並且不復制該類的可變狀態。

可以通過快速修復生成基本的 clone() 方法，它可以作為 Cloneable 類別所期望的正常執行的 clone() 方法的基礎。

範例：


  public class Data implements Cloneable {
    private String[] names;
  }

在套用快速修復後：


  public class Data implements Cloneable {
    private String[] names;

    @Override
    public Data clone() {
      try {
        Data clone = (Data) super.clone();
        // TODO：在此處複製可變狀態，因此克隆無法更改原始克隆的內部項 
         return clone;
      } catch (CloneNotSupportedException e) {
        throw new AssertionError();
      }
    }
  }

使用忽略由於繼承而可克隆的類別選項可以忽略因繼承自 Cloneable 類別而 Cloneable 的類別。

使用忽略需要 Cloneable 才能呼叫超類別的 clone() 方法的情況選項可以忽略因從超類別呼叫 clone() 方法而需要實作 Cloneable 的類別。
