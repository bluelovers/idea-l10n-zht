報告實現 Cloneable 接口但不重寫 clone() 方法的類。

這種類使用 clone() 的預設實現，它不是 public，而是 protected，並且不復制該類的可變狀態。

可以通過快速修復生成基本的 clone() 方法，它可以作為 Cloneable 類所期望的正常運行的 clone() 方法的基礎。

示例：


  public class Data implements Cloneable {
    private String[] names;
  }

在應用快速修復後：


  public class Data implements Cloneable {
    private String[] names;

    @Override
    public Data clone() {
      try {
        Data clone = (Data) super.clone();
        // TODO：在此處複製可變狀態，因此克隆無法更改原始克隆的內部項 
         return clone;
      } catch (CloneNotSupportedException e) {
        throw new AssertionError();
      }
    }
  }

使用忽略由於繼承而可克隆的類選項可以忽略因繼承自 Cloneable 類而 Cloneable 的類。

使用忽略需要 Cloneable 才能調用超類的 clone() 方法的情況選項可以忽略因從超類調用 clone() 方法而需要實現 Cloneable 的類。
