報告硬編碼 String 文字的任何實例。

硬編碼的 String 文字在國際化環境中可能是錯誤的。 此檢查不會報告空字串和僅包含空格的字串。 可以使用快速修復將字串文字轉換為 java.util.ResourceBundle.getString() 方法呼叫。

使用檢查的選項可進一步專門化此檢查是否應報告以下位置的字串：

  • 斷言語句，如 assert str.equals("message") 中
  • 異常構造函式呼叫，如 new Exception("message") 中
  • JUnit 斷言呼叫，如 assertEquals(str, "message") 中
  • 返回 String 的方法的唯一實參，如 getStringByKey("key") 中
  • 具有合法和現有類別名值的文字，如 Class.forName("java.lang.Object")
  • 具有合法和現有屬性鍵值的文字 bundle.getString("authentication.failed")
