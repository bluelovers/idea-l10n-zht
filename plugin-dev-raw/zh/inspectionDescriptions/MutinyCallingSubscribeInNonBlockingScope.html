報告 "reactive" 方法中的 subscribe() 呼叫。

返回 Publisher（包括 Uni 和 Multi）的方法不應直接呼叫 subscribe() 方法，因為它可能破壞反應式呼叫鏈。

不要使用 subscribe()，而應考慮使用組合運算符，例如 flatMap()、merge() 等。

範例：


  Multi<String> stringMulti(){
    Multi<String> multi = Multi.createFrom().item("abc");
    multi.subscribe(); // <- 在非阻塞上下文中阻塞 'subscribe' 呼叫
    return multi;
  }
