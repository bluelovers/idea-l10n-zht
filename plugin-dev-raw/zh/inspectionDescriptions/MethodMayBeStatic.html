報告可以安全設定為 static 的方法。 盡可能將方法設定為 static 可減少內存消耗並提高程式碼質量。

如果滿足以下條件，則方法可以為 static：

  • 不是 synchronized、native 或 abstract，
  • 不引用包含類中的任何非 static 方法和非 static 字段，
  • 不是覆寫，也不會在子類中被覆寫。

使用以下選項配置檢查：

  • 是否只報告 private 和 final 方法，從而提高此檢查的性能。
  • 是否忽略空方法。
  • 使用 Java 8 或更高版本時是否忽略接口中的預設方法。
  • 是否讓該快速修復在調用被設為 static 的方法時使用類引用取代實例限定符，即調用 myClass.m() 將被取代為 MyClass.m()。
