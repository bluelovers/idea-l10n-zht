報告覆寫超類方法但未使用 @java.lang.Override 註解的方法。

由於表明了意圖，具有 @java.lang.Override 的註解方法提高了程式碼可讀性。 此外，在覆寫方法的簽名與超類方法不符合時，編譯器會報錯。

範例：


    class X {
      public String toString() {
        return "hello world";
      }
    }

在套用快速修復後：


    class X {
      @Override
      public String toString() {
        return "hello world";
      }
    }

設定檢查：

  • 使用忽略 'equals()'、'hashCode()' 和 'toString()' 選項可以忽略這些 java.lang.Object 方法：equals()、hashCode() 和 toString()。 由於 @Override 註解而使得這些方法消失且程式碼不再編譯的風險相對較小。
  • 使用忽略匿名類中的方法選項可以忽略匿名類中的方法。
  • 禁用當方法的覆寫方法並非全部具有 '@Override' 註解時，高亮顯示該方法選項，可以僅就缺少 @Override 註解的方法發出警告，而不就其一個或多個後代缺少 @Override 註解的被覆寫方法發出警告。

僅當專案或模組的語言級別為 5 或更高時，此檢查才會報告。
