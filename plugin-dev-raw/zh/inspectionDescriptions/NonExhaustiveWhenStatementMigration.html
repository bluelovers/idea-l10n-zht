報告從 1.7 版開始會導致編譯錯誤的非詳盡 when 語句。

動機類型：

  • 需要勸阻/阻止的有問題/無意義的用法模式（例如反直覺的行為）
    • 程式碼容易出錯
  • 設計中的不一致（在不同的上下文中存在不同的處理方式）

影響類型：

  • 編譯。 一些過去可以編譯的程式碼將無法編譯
    • 在某些情況下，此類別程式碼可以無異常地工作
      • 某些此類別程式碼可以在沒有任何警告的情況下編譯

更多詳細資訊：KT-47709：預設情況下使帶有枚舉、sealed 和布爾主體的 when 語句詳盡

快速修復會新增缺少的 else -> {} 分支。

範例：


  sealed class Base {
      class A : Base()
      class B : Base()
  }

  fun test(base: Base) {
      when (base) {
          is Base.A -> ""
      }
  }

在套用快速修復後：


  sealed class Base {
      class A : Base()
      class B : Base()
  }

  fun test(base: Base) {
      when (base) {
          is Base.A -> ""
          else -> {}
      }
  }

僅當專案或模組的 Kotlin 語言級別為 1.6 或更高時，此檢查才會報告。
