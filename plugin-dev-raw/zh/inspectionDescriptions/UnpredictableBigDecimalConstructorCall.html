報告對接受 double 值的 BigDecimal 構造函數的調用。 這些構造函數生成正好等於提供的 double 值的 BigDecimal。 但是，由於 double 是以 IEEE 754 64 位雙精度二進制浮點格式進行編碼，確切值可能出乎意料。

例如，new BigDecimal(0.1) 會生成一個 BigDecimal 物件。 它的值為 0.1000000000000000055511151231257827021181583404541015625，這是最接近 0.1 的數字，可以表示為 double。 要獲取存儲的值與原始碼中編寫的值相同的 BigDecimal，請使用 new BigDecimal("0.1") 或 BigDecimal.valueOf(0.1)。

範例：


  class Constructor {
    void foo() {
      new BigDecimal(0.1);
    }
  }

在套用快速修復後：


  class Constructor {
    void foo() {
      new BigDecimal("0.1");
    }
  }
