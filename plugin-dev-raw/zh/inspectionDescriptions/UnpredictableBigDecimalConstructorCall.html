報告對接受 double 值的 BigDecimal 建構函式的呼叫。 這些建構函式生成的 BigDecimal 與提供的 double 值相等。 但是，由於 double 是以 IEEE 754 64 位雙精度二進制浮點格式進行編碼，確切值可能出乎意料。

例如，new BigDecimal(0.1) 會產生一個 BigDecimal 物件，值為 0.1000000000000000055511151231257827021181583404541015625，這是以 double 表示最接近 0.1 的數字。 要獲得包含期望值 0.1 的BigDecimal，請使用 new BigDecimal("0.1") 或 BigDecimal.valueOf(0.1)。

範例：


  class Constructor {
    void foo() {
      new BigDecimal(0.1);
    }
  }

在套用快速修復後：


  class Constructor {
    void foo() {
      new BigDecimal("0.1");
    }
  }
