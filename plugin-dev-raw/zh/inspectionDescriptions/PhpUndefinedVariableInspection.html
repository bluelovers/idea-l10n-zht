產生兩種類型的警告：
  • 未定義變數：在專案檔案、組態的 include 路徑或 PHP 預定義變數中找不到該變數的定義。
  • 變數可能還未定義：存在一條或多條路徑到達變數的用法行而此時變數還沒有定義。

使用以下選項來自定義檢查：

  • 報告變數可能未定義：清除複選框以停用 變數可能還未定義 警告。
  • 在全域空間中啟用檢查：預設情況下，對全域空間啟用檢查。 清除複選框以僅報告 函式本地作用域中的未定義變數。

  • 在當前檔案外搜尋變數定義：如果選中該選項，僅當當前檔案中沒有 include 或 require 語句以及變數的定義在專案中找不到時，IDE 才會將全域變數報告為未定義。 請注意，IDE 不檢查所包含檔案的實際內容，而只檢查是否存在 include 或 require 語句。

    否則，如果未選擇，IDE 僅在當前檔案中搜尋變數的定義，並且忽略 include 或 require 語句（如果有）。
