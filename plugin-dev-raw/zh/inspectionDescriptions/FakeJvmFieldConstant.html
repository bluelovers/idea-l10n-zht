報告用作 Java 註解實參的非 const Kotlin 屬性。 例如，使用 @JvmField 註解的屬性有一個可以在編譯時評估的初始設定式，它有一個基元類型或 String 類型。

此類屬性在 Kotlin 1.1-1.2 中的字節碼中具有 ConstantValue 屬性。 此屬性允許 javac 折疊相應欄位的用法並在註解中使用該欄位。 在混合 Java/Kotlin 程式碼中單獨或增量編譯的情況下，這可能導致錯誤行為。 此行為在 Kotlin 1.3 中可能發生變化（不再有 ConstantValue 特性）。 建議操作：將 @JvmField 註解取代為相關 Kotlin 屬性上的 const 修飾符或將其內聯。
