報告不是 const 並用作 Java 註解實參的 Kotlin 屬性。

例如，具有 @JvmField 註解的屬性有一個可以在編譯時評估的初始設定式，它有一個基元類型或 String 類型。

此類別屬性在 Kotlin 1.1-1.2 中的位元組碼中具有 ConstantValue 特性。 此特性允許 javac 折疊相應欄位的用法並在註解中使用該欄位。 在混合 Java/Kotlin 程式碼中單獨或增量編譯的情況下，這可能導致錯誤行為。 此行為在 Kotlin 1.3 中可能發生變化（不再有 ConstantValue 特性）。

範例：

foo.kt 檔案中的 Kotlin 程式碼：


  annotation class Ann(val s: String)
  @JvmField val importantString = "important"

Java 程式碼：


  public class JavaUser {
      // 這很危險
      @Ann(s = FooKt.importantString)
      public void foo() {}
  }

要修正此問題，請將 @JvmField 註解取代為相關 Kotlin 屬性上的 const 修飾符或將其內聯。
