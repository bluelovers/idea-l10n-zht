報告雙重檢查鎖定。

雙重檢查鎖定會嘗試以執行緒安全的方式按需初始化欄位，同時避免同步的開銷。 遺憾的是，在未被宣告為 volatile 的欄位上使用時，它不具備執行緒安全。 在使用 Java 1.4 或更早的版本時，雙重檢查鎖定即便對 volatile 欄位也不起作用。 閱讀上面的鏈接文章，了解該問題的詳細說明。

不正確的雙重檢查鎖定範例：


  class Foo {
    private Helper helper = null;
    public Helper getHelper() {
      if (helper == null)
        synchronized(this) {
          if (helper == null) helper = new Helper();
        }
        return helper;
      }
    }
    // 其他函式和成員…
  }
