報告無法在組態的資料來源中解析的 Persistence ORM XML 檔案描述子並建議重新整理資料來源或分配不同的資料來源。

範例資料庫表：


  CREATE TABLE DATA_TABLE (
    ID INTEGER,
    DATA VARCHAR(100)
  );

範例 JSP 檔案：


  <?xml version="1.0" encoding="UTF-8"?>
  <entity-mappings xmlns="http://java.sun.com/xml/ns/persistence/orm" version="1.0">
    <entity name="JavaEntity" class="JavaEntity">
        <table name="UNKNOWN_TABLE"/> // 錯誤：未解析的表
        <attributes>
          <id name="id">
            <column name="ID" nullable="false"/>
          </id>
          <basic name="UNKNOWN_COLUMN"> // 錯誤：未解析的列
            <column name="UNKNOWN_COLUMN" length="12"/>
          </basic>
        </attributes>
    </entity>
  </entity-mappings>

注意：資料來源可以通過 `分配資料來源` 動作進行組態
