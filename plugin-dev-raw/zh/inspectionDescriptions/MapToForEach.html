報告結果被忽略且可被取代為 forEach 的 map 呼叫。

以這種方式使用 map 具有誤導性，因為 map 旨在轉換集合並返回結果。 當它的返回值未使用時，呼叫會建立不必要的分配並降低程式碼可讀性。 在這種情況下，應改用 forEach，因為它清楚地表達了具有副作用的迭代。

該快速修復會將 map 呼叫取代為 forEach 呼叫。

範例：


fun foo(list: List<String>) {
 list.map { print(it) } // 已報告

 val x = list.map { it.length } // 未報告：結果已使用
}

在套用快速修復後：


fun foo(list: List<String>) {
 list.forEach { print(it) }

 val x = list.map { it.length } // 未報告：結果已使用
}
