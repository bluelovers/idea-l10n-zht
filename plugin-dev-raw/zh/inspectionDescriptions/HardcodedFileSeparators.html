報告字串或字符文字中的正斜杠 (/) 或反斜杠 (\)。 這些字符通常用作文件分隔符，如果它們被硬編碼，則可移植性可能會受到影響。

該檢查不會報告轉義序列內的反斜杠和緊跟在 '<' 字符之後或緊接在 '>' 字符之前的正斜杠，因為它們通常表示 XML 或 HTML 標記而不是文件名。 表示 java.util.TimeZone ID 的字串、作為有效正則表達式的字串或與 IANA 註冊的 MIME 媒體類型等效的字串也不會被報告。

範例：


  new File("C:\\Users\\Name");

使用該選項可將 example/* 包含在已識別的媒體類型集中。 通常，在範例之外（例如在 Content-Type 標頭中）使用 example/* MIME 媒體類型是錯誤的。
