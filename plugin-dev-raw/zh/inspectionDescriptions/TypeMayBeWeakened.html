報告可以更改為更抽象（更弱）類型的變數和方法返回類型。 這使程式碼更抽象，因此可重用性更高。

範例：


  // 參數類型可以削弱為 java.util.List
  void processList(ArrayList<String> list) {
    if (list.isEmpty()) return;
    System.out.println("Processing");
    for (String s : list) {
      System.out.println("String: " + s);
    }
  }

啟用下面的使用右側類型複選框可防止在右側並非類型轉換或新表達式時削弱賦值的左側。 在變數中存儲方法呼叫結果時，保留方法呼叫結果的類型不會不必要地削弱它，反而會很有用。

啟用下面的使用參數化類型複選框可在求值的物件用作參數類型為 java.lang.Object 的集合方法的實參時，將集合的參數化類型用作最弱的類型。 在將物件傳遞給以下集合方法時，使用此選項可以防止削弱為 Object： get()、remove()、contains()、indexOf()、lastIndexOf()、containsKey() 和 containsValue()。

啟用下面的不削弱為 Object 複選框可以指定是否應將類型削弱為 java.lang.Object。 削弱為 java.lang.Object 很少有用。

啟用下面的僅削弱為接口複選框，可僅在該類型可以削弱為接口類型時才報告問題。

啟用下面的不削弱返回類型複選框，可防止在返回類型可能被削弱時報告問題。 只分析變數。

啟用下面的不建議削弱宣告為 'var' 的變數複選框，可防止報告使用 'var' 關鍵字宣告的區域變數 (Java 10+)

停止類別意在防止類別削弱至低於停止類別的程度，即便可能出現這種情況，也能予以防止。 在某些情況下，這可能會提高可讀性。
