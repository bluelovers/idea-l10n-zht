報告從 catch 塊內部拋出但不“包裝”所捕獲異常的異常。

當異常被拋出以響應異常時，包裝初始異常可防止丟失有價值的上下文資訊，如堆疊幀和行號。

範例：


  ...
  catch (IOException e) {
    closeAllConnections();
    throw new ConnectException("Connection problem."); // 警告：'catch' 塊內的 'throw' 忽略捕獲的異常 'e'
  }

組態檢查：

  • 使用如果使用異常方法呼叫的結果，則忽略選項可指示檢查是否應忽略實參為原始異常方法呼叫結果的異常，例如 getMessage()。
  • 使用如果拋出的異常無法包裝異常，則忽略選項可忽略在沒有接受 Throwable 子句的建構函式的情況下拋出異常的 throw 語句。
