報告用來初始化的 class 字面量所屬的類別與 Logger 所屬的類別不同的 Logger 實例。 從另一個類別複製貼上某些程式碼時，很容易發生這種情況，並可能導致在意外類別下記錄事件，並導致篩選器套用不正確。

提供了一個快速修復來用周圍類別中的字面量取代外類別字面量。

範例：


  public class Paramount {
    protected static final Logger LOG = Logger.getLogger(Critical.class);

    // …其他欄位和方法
  }

在套用快速修復後：


  public class Paramount {
    protected static final Logger LOG = Logger.getLogger(Paramount.class);

    // …其他欄位和方法
  }

組態檢查：

  • 使用該表可以指定此檢查識別的記錄器工廠類別和記錄器工廠方法。
  • 使用忽略使用超類別初始化的記錄器選項可以忽略使用包含該記錄器的類別的超類別初始化的記錄器。
  • 使用忽略非 public 類別中的記錄器，僅對 public 類別中的記錄器發出警告。
  • 使用忽略未在 final 欄位中初始化的記錄器可以僅報告在 final 欄位中初始化的記錄器，其他情況將被忽略。
