報告最後的 else 分支為點限定或二元表達式一部分的級聯 if 表達式。

這可能導致意外行為，因為只有最內層的 if-else 表達式成為外部表達式的接收器或左動作數，這可能與預期邏輯不一致。

範例：


fun printNumberSign(num: Int) {
    if (num > 0) {
        "positive"
    } else if (num < 0) {
        "negative"
    } else {
        "zero"
    }.let { print(it) }
}

fun main() {
    printNumberSign(1) // 未列印任何內容，因為 'let' 的接收者僅為內層的 'if-else' 表達式
}

套用第一個快速修復（轉換為 when）後：


fun printNumberSign(num: Int) {
    when {
        num > 0 -> {
            "positive"
        }

        num < 0 -> {
            "negative"
        }

        else -> {
            "zero"
        }
    }.let { print(it) }
}

fun main() {
    printNumberSign(1) // 列印“positive”
}

或者，第二個快速修復會新增明確的大括號以保持原始語意，同時將嵌套的 else 分支設為顯式：


fun printNumberSign(num: Int) {
    if (num > 0) {
        "positive"
    } else {
        if (num < 0) {
            "negative"
        } else {
            "zero"
        }.let { print(it) }
    }
}

fun main() {
    printNumberSign(1) // 未列印任何內容，但程式碼更清晰
}
