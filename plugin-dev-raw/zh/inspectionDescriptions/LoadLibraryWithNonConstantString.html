報告對java.lang.System.loadLibrary()、java.lang.System.load()、java.lang.Runtime.loadLibrary() 和 java.lang.Runtime.load() 的調用，它們會將動態構造的字串作為庫的名稱。

構造的庫名稱字串是安全漏洞的常見來源。 預設情況下，此檢查會忽略編譯時常量。

範例：


  void test(int i) {
    System.loadLibrary("foo" + i);
  }

使用檢查設定可以將任何 static final 欄位視為常數。 請小心，因為啟用該選項時，像下面這樣的字串將被忽略： 


  private static final String LIBRARY = getUserInput();
