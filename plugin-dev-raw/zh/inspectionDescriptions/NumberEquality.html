報告使用 == 或 != 而不是 equals() 來測試 Number 相等性的程式碼。

使用自動裝箱時很容易犯這樣的錯誤：即不比較兩個基元，而是比較包裝器類型的兩個實例，例如使用 Integer 而不是 int。

範例：

  void foo(Integer a, Integer b) {
    final boolean bool = a == b;
  }

如果已知 a 為非 null，則可以安全地套用“不安全”的快速修復，並獲得類似於如下所示的結果：

  void foo(Integer a, Integer b) {
    final boolean bool = a.equals(b);
  }
