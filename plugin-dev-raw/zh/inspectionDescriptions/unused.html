報告指定檢查範圍中未使用或無法從入口點到達的類、方法或欄位。

入口點可以是主方法、測試、指定範圍外提及的類、可從 module-info.java 存取的類，等等。 還可以使用名稱模式或註解來設定自訂入口點。

範例：


  public class Department {
    private Organization myOrganization;
  }

在本例中，Department 顯式引用 Organization，但如果 Department 類本身未使用，那麼該檢查將報告這兩個類。

該檢查還會報告其方法、所有方法實作和覆寫器未使用的參數，以及已宣告但未使用的局部變數。

注意：在編輯器中高亮顯示程式碼期間可能不會報告某些未使用的成員。 由於性能原因，僅當非 private 成員的名稱很少出現在專案中時，才會對其進行檢查。 要查看所有結果，請從主選單中選擇程式碼 | 檢查程式碼或程式碼 | 分析程式碼 | 按名稱執行檢查來執行檢查。

使用下面的可見性設定可以設定要報告的成員。 例如，設定報告 private 方法只表明將報告 private 內部類的 public 方法，頂層類的 protected 方法將被忽略。

使用入口點頁籤可以設定檢查執行期間要考慮的入口點。

在檢查結果準備就緒後，可以手動添加入口點。

如果您的程式碼使用不受支援的框架，可以選擇以下幾個選項：

  • 如果框架依賴於註解，使用註解…按鈕可設定框架註解。
  • 如果框架不依賴於註解，可嘗試設定框架應有的類名模式。

這樣，框架內部項可存取的註解程式碼將被視為已使用。
