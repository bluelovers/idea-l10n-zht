報告未使用或無法從入口點存取的類別、方法或欄位。

入口點可以是 main 方法、測試、來自指定範圍之外的類別、可從 module-info.java 存取的類別，等等。 可以通過使用名稱模式或註解來組態自訂入口點。

範例：


  public class Department {
    private Organization myOrganization;
  }

在本例中，Department 顯式參照 Organization，但如果 Department 類別本身未使用，那麼該檢查將報告這兩個類別。

該檢查還會報告其方法、所有方法實作和覆寫器未使用的參數，以及已宣告但未使用的區域變數。

注意：在編輯器中醒目提示程式碼期間可能不會報告某些未使用的成員。 由於性能原因，僅當非 private 成員的名稱很少出現在專案中時，才會對其進行檢查。 要檢視所有結果，請從主選單中選擇程式碼 | 檢查程式碼或程式碼 | 分析程式碼 | 按名稱執行檢查來執行檢查。

使用下面的可見性設定可以組態要報告的成員。 例如，組態報告 private 方法只表明將報告 private 內部類別的 public 方法，頂層類別的 protected 方法將被忽略。

使用入口點頁籤可以組態檢查執行期間要考慮的入口點。

在檢查結果準備就緒後，可以手動新增入口點。

如果您的程式碼使用不受支援的框架，可以選擇以下幾個選項：

  • 如果框架依賴於註解，使用註解…按鈕可組態框架註解。
  • 如果框架不依賴於註解，可嘗試組態框架應有的類別名模式。

這樣，框架內部項可存取的註解程式碼將被視為已使用。
