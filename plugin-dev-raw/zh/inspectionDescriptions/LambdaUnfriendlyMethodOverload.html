報告採用的函數接口存在抽象方法簽名衝突的多載方法。

此類多載會引入歧義，並要求調用方將 lambda 轉換為特定類型或顯式指定 lambda 參數類型。 最好為多載方法賦予不同的名稱，以消除歧義。

範例：


  interface MyExecutor {
    void execute(Supplier<?> supplier);
    void execute(Callable<?> callable);
  }

這裡的 Supplier 和 Callable 為函數接口，其單抽象方法不接受任何參數並返回非 void 值。 因此，除非使用顯式轉換，否則無法在調用點推斷 lambda 的類型。
