報告 SQL 程式碼中不明確的引用。

例如，當名稱同時引用表列和例程參數時。 由於違反直觀的解析邏輯，此類別程式碼的執行可能會導致錯誤或意外結果。 通常，範圍更局部的名稱具有更高的優先級。

範例（PostgreSQL）：

CREATE TABLE foo
(
    id   INT,
    name VARCHAR(5)
);
CREATE FUNCTION func(name VARCHAR(5)) RETURNS INT AS
$$
DECLARE
    b INT;
BEGIN
      -- `name` 不明確，因為它被用作列名和參數
    SELECT COUNT(*) INTO b FROM foo t WHERE t.name = name;
    RETURN b;
END;
$$ LANGUAGE plpgsql;

在 PostgreSQL 中，您可以使用 #variable_conflict 指令來明確指定正確的引用。 例如，使用 #variable_conflict use_column 引用列名，或使用 #variable_conflict use_variable 引用參數。

CREATE TABLE foo
(
    id   INT,
    name VARCHAR(5)
);
CREATE FUNCTION func(name VARCHAR(5)) RETURNS INT AS
$$
    #variable_conflict use_column
DECLARE
    b INT;
BEGIN
    SELECT COUNT(*) INTO b FROM foo t WHERE t.name = name;
    RETURN b;
END;
$$ LANGUAGE plpgsql;
