報告可以使用有界萬用字元的通用方法參數。

範例：


  void process(Consumer<Number> consumer);

應取代為：


  void process(Consumer<? super Number> consumer);

此方法簽名更靈活，因為它接受更多類型：不僅是 Consumer<Number>，還有 Consumer<Object>。

同樣，在協變位置輸入參數：


  T produce(Producer<T> p);

應取代為：


  T produce(Producer<? extends T> p);

參照 Effective Java 第三版中的 Joshua Bloch：

第 31 項：使用有界萬用字元增加 API 靈活性

在 API 中使用萬用字元類型雖然麻煩，但會使 API 更加靈活。 如果您編寫的庫將被廣泛使用，則應強制正確使用萬用字元類型。 記住基本規則：生產者-extends, 使用者-super (PECS)。 還要記住，所有 Comparable 和 Comparator 都是消費者。

使用檢查選項可以切換以下各項的報告：

  • 不變類別。 一個不變類別的範例是 java.util.List<T>，因為它既接受值（通過 List.add(T) 方法）又生成值（通過 T List.get() 方法）。

    另一方面，contravariant 類別只接收值，例如，只有方法 accept(T) 的 java.util.function.Consumer<T>。 類似地，covariant 類別只產生值，例如，只有方法 T get() 的 java.util.function.Supplier<T>。

    人們經常在協變/逆變類別中使用有界萬用字元，但避免在不變類別中使用萬用字元，例如 void process(List<? extends T> l)。 停用此選項可以忽略這種不變類別並保留它們的嚴格類型，例如 void process(List<T> l)。
