檢測 Fleet 程式碼庫中不當使用 Kotlin 序列化方法的情況，這些方法缺少顯式的序列化/反序列化策略參數。

此檢查可以識別對 JSON 序列化方法的呼叫，這些呼叫在未適當參數化時可能導致執行時意外反射。

該檢查會專門檢查對 kotlinx.serialization.json.Json 軟體套件中以下方法的呼叫： 


decodeFromString
decodeFromJsonElement
encodeToString
encodeToJsonElement

這些方法應始終使用顯式序列化/反序列化策略參數進行呼叫，以避免依賴於反射，這可能會在 Fleet 的多平台環境中引發問題。

有問題的程式碼範例：


Json.encodeToString(myObject) // 缺少 SerializationStrategy

正確用法：


Json.encodeToString(MyType.serializer(), myObject)
