報告未篩選的 for-in 迴圈。

使用此結構不僅會處理物件自身的屬性，還會處理物件原型的屬性。 在某些特定情況下可能出人意料，例如，在複製或修改所有屬性的實用程序方法中 或者 Object 的原型可能被錯誤修改時。 例如，以下程式碼將列印 42 和 myMethod： 


Object.prototype.myMethod = function myMethod() {};
let a = { foo: 42 };
for (let i in a) {
  console.log(a[i]);
}

建議將整個迴圈取代為 Object.keys() 方法，或新增 hasOwnProperty() 檢查。 套用快速修復後，程式碼如下所示：


for (let i in a) {
  if (a.hasOwnProperty(i)) {
    console.log(a[i]);
  }
}
