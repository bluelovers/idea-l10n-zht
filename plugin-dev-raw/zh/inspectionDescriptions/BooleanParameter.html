報告接受 boolean 參數的 public 方法。

如果某個 public 方法（API 的一部分）不是 setter，則向該方法添加 boolean 參數幾乎始終是一種不好的做法。 使用此類方法讀取程式碼時，如果不查看原始碼或文檔，會很難理解 boolean 的含義。

此問題也被稱為布爾陷阱。 boolean 參數通常可以取代為 enum。

範例：


  // 警告：在查看對此方法的
  // 調用時，很難理解
  // 布爾參數的含義
  public boolean setPermission(File f,
                               int access,
                               boolean enable,
                               boolean ownerOnly) {
    // ...
  }

使用僅報告具有多個布爾參數的方法選項可以僅在一個方法包含多個布爾參數時發出警告。
