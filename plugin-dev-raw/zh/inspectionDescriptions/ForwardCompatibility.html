報告未來 Java 版本中可能無法編譯的 Java 程式碼結構。

將報告以下問題：

  • 使用 assert、enum 或 _ 作為關鍵字
  • 使用 var、yield 或 record 受限關鍵字作為類型名稱
  • 對 yield 方法的非限定呼叫
  • module-info.java 內部的 requires java.base 語句的修飾符
  • import 語句之間的冗餘分號

範例：


  // 這個以前合法的類別無法在 Java 14 中編譯，
  // 因為 'yield' 變成了一個受限的關鍵字。
  public class yield {}

及時解決這些問題，可以簡化向未來 Java 版本的遷移。
