報告可以不使用更詳細或效率較低的結構，而是使用 static Integer.compare() 方法或類似方法的情況。

如果 x 和 y 已是裝箱的整數，則建議使用 x.compareTo(y)。

範例：


  public int compare(int x, int y) {
    return x > y ? 1 : x < y ? -1 : 0;
  }

在套用快速修復後：


  public int compare(int x, int y) {
    return Integer.compare(x, y);
  }

請注意，Double.compare 和 Float.compare 會略微更改程式碼語意。 特別是，它們會使 -0.0 和 0.0 可以區分（Double.compare(-0.0, 0.0) 生成 -1）。 此外，它們會一致地處理 NaN 值。 在大多數情況下，這種語意更改實際上會改進程式碼。 如果您的情況無法接受語意更改，請使用複選框禁用此項對浮點數的檢查。
