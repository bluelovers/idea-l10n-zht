報告在 java.lang.Class 實例上呼叫的 getClass() 方法。

這通常是一個錯誤，因為結果總是等同於 Class.class。 如果是錯誤，那麼最好移除 getClass() 呼叫並直接使用限定符。 如果行為是故意的，最好顯式編寫 Class.class 以避免混淆。

範例：


  void test(Class<?> clazz) {
    String name = clazz.getClass().getName();
  }

在套用可能的快速修復之一後：


  void test(Class<?> clazz) {
    String name = clazz.getName();
  }
