檢測可解釋為 OpenAPI/Swagger 規範一部分的 JSON 檔案。

此類別檔案不包含顯式規範特性，但它們參照自附近的主規範檔案。

下面的範例包含位於同一目錄中的兩個檔案。 第一個是一個普通的主規範檔案。 第二個參照自第一個，因此建議將其視為一個規範。

主規範檔案 openapi.yaml：


openapi: 3.1.0
components:
  schemas:
    CustomSchema:
      description: Custom schema object
      properties:
        foo:
          $ref: 'common.components.json#/components/schemas/CommonSchema'

規範檔案候選項 common.components.json：


{
  "components": { // 'Mark file as OpenAPI specification' highlighting
    "schemas": {
      "CommonSchema": {
        "description": "Common schema object reused in several specifications"
      }
    }
  }
}
