報告對無法為之找到相應 Condition.signal() 或 Condition.signalAll() 呼叫的 Condition.await() 的呼叫。

在沒有相應 Condition.signal() 的執行緒中呼叫 Condition.await() 可能導致該線程被停用，直到它被中斷或發生“可疑喚醒”為止。

此檢查僅報告以當前類別的目標欄位為目標的呼叫。

範例：


  class Queue {
    private final Condition isEmpty = ...;

    void add(Object elem) {
      // ...
      // isEmpty.signal();
      // ...
    }

    void remove(Object elem) throws InterruptedException {
      // ...
      isEmpty.await(); // 'await()' 不包含相應的 'signal()'/'signalAll()' 呼叫
      // ...
    }
  }
