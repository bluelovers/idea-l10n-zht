報告在屬性存取器上重複使用非 @Repeatable 註解的情況。

由於多次使用非 @Repeatable 註解，這兩種註解的使用都會出現在位元組碼中，從而導致反射呼叫中的歧義。

從 Kotlin 1.4 開始，必須將註解標記為 @Repeatable 或不重複註解，否則會導致編譯錯誤。

範例：


  annotation class Foo(val x: Int)

  @get:Foo(10)
  val a: String
      @Foo(20) get() = "foo" // 註解重複兩次，但未被標記為 @Repeatable

僅當專案或模組的 Kotlin 語言級別為 1.4 或更高時，此檢查才會報告。
