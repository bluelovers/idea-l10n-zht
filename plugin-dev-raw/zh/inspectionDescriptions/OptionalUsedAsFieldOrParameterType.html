報告 java.util.Optional<T>、java.util.OptionalDouble、java.util.OptionalInt、java.util.OptionalLong 或 com.google.common.base.Optional 用為欄位或參數的類型的任何情況。

Optional 設計為在需要一個明確的方式來表示 "沒有結果"的情況下，為庫方法的返回類型提供有限的機制。

如果類別需要 Serializable，則使用 java.util.Optional 類型的欄位也有問題，因為 java.util.Optional 不可序列化。

範例：


  class MyClass {
    Optional<String> name; // Optional 欄位

    // Optional 參數
    void setName(Optional<String> name) {
      this.name = name;
    }
  }
