報告空的或可能忽略異常的 catch 塊。

雖然偶爾是有意為之，但空的 catch 塊可能導致偵錯復雜化。 此外，忽略 catch 參數可能是錯誤的。 最後，靜態程式碼分析器會報告是否檢測到 catch 塊可能無提示地忽略重要的 VM 異常，如 NullPointerException。 忽略此類別異常（不記錄或重新拋出）可能會隱藏錯誤。

該檢查不會報告任何命名為 ignore 或 ignored 的 catch 參數。 相反，它將對真正在使用的命名為 ignore 或 ignored 的 catch 參數發出警告。 此外，該檢查不會報告命名為 expected 或 ok 的測試源中的 catch 參數。

您可以使用快速修復將異常名稱更改為 ignored。 對於空的 catch 塊，建議使用其他快速修復來生成 catch 主體。 您可以在設定 | 編輯器 | 檔案和程式碼模板中的“程式碼”頁籤上修改 "Catch Statement Body" 模板。

範例：


  try {
    throwingMethod();
  } catch (IOException ex) {

  }

在套用快速修復後：


  try {
    System.out.println(System.in.read());
  } catch (IOException ignored) {

  }

組態檢查：

  • 使用當 'catch' 塊包含註釋時不發出警告選項可以忽略包含註釋的 catch 塊。
  • 使用當 'catch' 塊不為空時不發出警告選項可以忽略其中包含語句或註釋但又不使用變數本身的 catch 塊。
  • 使用當名稱為 'ignore(d)' 的異常實際上未被忽略時不發出警告選項可以忽略使用中名稱為 ignored 的變數。
