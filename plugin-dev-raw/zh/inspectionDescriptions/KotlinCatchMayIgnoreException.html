報告空的或可能忽略異常的 catch 塊。

雖然偶爾是有意為之，但空的 catch 塊可能導致偵錯復雜化。 此外，忽略 catch 參數可能是錯誤的。

該檢查不會報告任何命名為 ignore、ignored 或 _ 的 catch 參數。

您可以使用快速修復將異常名稱更改為 _。

範例：


  try {
    throwingMethod()
  } catch (ex: IOException) {

  }

在套用快速修復後：


  try {
    throwingMethod()
  } catch (_: IOException) {

  }
使用當 'catch' 塊包含註釋時不發出警告選項可以忽略包含註釋的 catch 塊。
