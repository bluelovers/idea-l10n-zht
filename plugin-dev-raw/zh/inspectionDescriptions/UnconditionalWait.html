報告在同步上下文中無條件呼叫的 wait()。

wait() 通常用於在某個條件為 true 前阻塞執行緒。 如果是無條件呼叫 wait()，它通常表示在獲取鎖定之前檢查了條件。 在這種情況下可能會發生資料競爭，條件在得到檢查和獲取鎖定之間變為 true。

雖然此檢查發現的結構未必不正確，但肯定值得檢查。

範例：


  class Bar {
      void foo() throws InterruptedException {
          synchronized (this) {
              wait(); // 警告
          }
      }
  }
