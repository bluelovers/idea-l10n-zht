報告陣列或可變類型（例如 Collection、Date、Map、Calendar 等）欄位的返回值或從方法參數到陣列或可變類型欄位的賦值。

由於這種類型是可變的，此結構可能會導致來自所屬類別之外的物件狀態發生意外修改。 儘管出於性能原因，此結構可能很有用，但它本質上很容易出現錯誤。

報告了以下可變類型：

  • java.util.Date
  • java.util.Calendar
  • java.util.Collection
  • java.util.Map
  • com.google.common.collect.Multimap
  • com.google.common.collect.Table

該快速修復會為陣列新增對欄位的 .clone() 方法的呼叫，或使用不可修改的集合包裝器。

範例：


  import java.util.*;
  class Log {
    private String[] messages = {"one", "two", "three"};
    private Map<String, String> map = new HashMap<>();

    String[] getMessages() {
      return messages; // 警告：返回 String[] 欄位 'messages'
    }

    Map<String, String> mapping() {
      return map; // 警告：返回 Map<String, String> 欄位 'map'
    }
  }

在套用快速修復後：


  import java.util.*;
  class Log {
    String[] messages = {"one", "two", "three"};
    private Map<String, String> map = new HashMap<>();

    String[] getMessages() {
      return messages.clone();
    }

    Map<String, String> mapping() {
      return Collections.unmodifiableMap(map);
    }
  }

使用忽略 private 方法中的賦值和返回值選項可忽略 private方法中的賦值和返回值。
