報告對 String.startsWith() 和 String.endsWith() 的呼叫，其中單字元字串文字會作為實參傳遞。

建議通過快速修復，使用更高效實作的 String.charAt() 取代此類別呼叫。

然而，由於額外的非零長度檢查，此類別更改的性能增益很小並且會降低程式碼的可讀性，建議僅在緊密迴圈內應用該快速修復。

此檢查適用於 Java ME 和其他資源高度受限的環境。 不加考慮地套用此檢查的結果可能會對程式碼清晰度和設計產生負面影響。

範例：


  boolean startsWithX(String s) {
    return s.startsWith("x");
  }

在套用快速修復後：


  boolean startsWithX(String s) {
    return !s.isEmpty() && s.charAt(0) == 'x';
  }
