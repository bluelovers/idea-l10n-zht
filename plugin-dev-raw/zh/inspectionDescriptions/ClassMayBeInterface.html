報告可以轉換為接口的 abstract 類別。

最好使用接口而不是類別，因為 Java 不支持多類別繼承，而一個類別可以實作多個接口。

如果一個類別沒有超類別（Object 以外）、只有 public static final 欄位、public abstract 方法和 public 內部類別，則可以將其轉換為接口。

範例：


abstract class Example {
    public static final int MY_CONST = 42;
    public abstract void foo();
}

class Inheritor extends Example {
    @Override
    public void foo() {
        System.out.println(MY_CONST);
    }
}

在套用快速修復後：


interface Example {
    int MY_CONST = 42;
    void foo();
}

class Inheritor implements Example {
    @Override
    public void foo() {
        System.out.println(MY_CONST);
    }
}

組態檢查：

使用 Java 8 時報告包含非 abstract 方法的類別 選項可用於僅報告其 static 和非 abstract 方法可以轉換為 default 方法的類別（僅適用於語言級別為 8 或更高時）。
