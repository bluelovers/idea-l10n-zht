報告在掛起上下文中可能存在問題的隱式 CoroutineScope 接收器存取。

當一個掛起函式或 Lambda 從外部上下文捕獲隱式 CoroutineScope 接收器時，可能會導致意外行為。 這是因為捕獲的作用域在掛起函式仍在執行時可能會被取消或完成。

此檢查可以檢測程式碼在掛起函式或 Lambda 中隱式存取 CoroutineScope 接收器的情況， 隱式存取會建立對外部作用域的潛在危險相依。

範例：


class MyClass {
    fun CoroutineScope.launchJobs() {
        launch { // 可以：直接在擴展函式中使用
            doSomething()
        }
        
        suspendingFunction { // 警告：可疑的隱式 'CoroutineScope' 接收器存取
            launch { // launch 呼叫使用外部上下文中的隱式 CoroutineScope
                doSomethingElse()
            }
        }
    }
}

選項:

  • 檢測 CoroutineScope 子類型 – 啟用後，該檢查還將檢測對 CoroutineScope 的子類型的隱式存取

要解決此問題，您可以：

  • 使用 this@receiverLabel.method() 明確指定接收器
  • 將作用域作為參數傳遞給掛起函式
  • 使用 coroutineScope { } 建置器建立一個與掛起函式的生命周期綁定的子作用域
