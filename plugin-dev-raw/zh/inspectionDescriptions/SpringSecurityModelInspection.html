報告 Spring Security 基於 XML 的上下文的問題：

  • 未解析的 Bean 參照
  • 缺少必需的標記或特性
  • 屬性類型不正確
  • enum 屬性不一致
  • 參照 Bean 的類型不正確

也報告以下問題：

  • <https> 組態錯誤
    • 帶 '/**' 模式的 <http> 必須位於最後
    • 不允許使用多個無 'pattern' 的 <http> 元素
  • <http> 組態錯誤
    • 如果 security='none'，則為空標記
  • <user-service>
    • 必須具有特性 'properties' 或 <user> 元素列表
  • <ldap-server>
    • 使用 'manager-dn' 時需要 'manager-password'
    • 必須具有 'user-context-mapper-ref' 或 'user-details-class'
  • <authentication-provider>
    • 僅允許使用一種使用者服務
  • <global-method-security>
    • 'pre-post-annotations' 的值必須為 'enabled' 才能使用 <pre-post-annotation-handling>
    • 無法為 <protect-pointcut> 和 <intercept-url> 使用值為 'aspectj' 的 'mode'
    • 僅對父 <http> 允許 'requires-channel' 特性
  • <protect>
    • 不建議將 <protect> 與 <global-method-security> 混用
