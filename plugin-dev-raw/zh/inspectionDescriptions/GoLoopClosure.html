報告在 defer 和 go 語句中的 func 文字內對迴圈變數的參照。 此類別變數可能具有意外值，因為它們沒有被複製到 func 文字中，並且 defer 和 go 中的 func 文字不會立即執行。

有關閉包和 goroutine 的更多資訊，請參閱 go.dev 上的 What happens with closures running as goroutines? 。

範例：

for _, v := range []string{"a", "b", "c"} {
  go func() {
    fmt.Println(v) // 輸出可能會是 `c c c`，而不是 `a b c`。
  }()
}

在套用快速修復後：

for _, v := range []string{"a", "b", "c"} {
  v := v // `v` 現已被複製
  go func() {
    fmt.Println(v)
  }()
}

請注意，僅當 defer 和 go 語句是迴圈主體中的最後一個語句時，分析器才會檢查它們。 否則，分析可能會產生錯誤的檢測結果。
