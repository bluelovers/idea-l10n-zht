報告 this 或 class 表達式的同步。 報告的結構包括 synchronized 塊以及對 wait()、notify() 或 notifyAll() 的呼叫。

同步 this 或 class 表達式可能並非好主意，原因有幾點：

  1. 它使同步成為類別的外部接口的一部分，導致將來很難更改為其他鎖定機制，
  2. 難以追蹤究竟是誰鎖定了給定物件，
  3. 使得故意為之或在子類別化時很容易意外發生的拒絕服務攻擊成為可能。

替代做法是考慮在 private final 鎖定物件上同步，可以完全控制對該物件的存取。

範例：


  public void print() {
    synchronized(this) { // 警告：對 'this' 的鎖定動作可能會產生不可預見的副作用
      System.out.println("synchronized");
    }
  }
