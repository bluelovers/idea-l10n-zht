報告可以取代為 lambda 表達式的匿名類別。

範例：


  new Thread(new Runnable() {
    @Override
    public void run() {
      // 執行執行緒
    }
  });

在套用快速修復後：


  new Thread(() -> {
    // 執行執行緒
  });

請注意，如果將匿名類別轉換為無狀態 lambda，則 Java Runtime 可以在後續呼叫期間重用相同的 lambda 物件。 另一方面，當使用匿名類別時，每次都會建立單獨的物件。 因此，在極少數情況下，套用該快速修復可能會導致語意發生變化，例如，當匿名類別實例用作 HashMap 鍵時。

Java 1.7 和更早版本的 JVM 不支持 lambda 語法。

使用在接口未使用 @FunctionalInterface 註解時報告選項可忽略匿名類別實作沒有 @FunctionalInterface 註解的接口的情況。
