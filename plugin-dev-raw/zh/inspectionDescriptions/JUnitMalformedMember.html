報告格式錯誤且可能無法被 JUnit 框架識別的 JUnit 成員宣告。 本次檢查報告的問題如下：
  • 使用 @RegisterExtension 註解的欄位，它們的類型錯誤或者在需要時未宣告為 static。
  • 使用 @Nested 註解的 static 內部類。
  • 在沒有源的情況下定義的參數化測試。
  • 具有未知、非 static 或無實參目標的 MethodSource 的參數化測試。
  • 具有無法轉換為方法參數的 ValueSource 或 EnumSource 的參數化測試。
  • 同時使用 `@Test`、`@ParameterizedTest` 或 `@RepeatedTest` 註解的測試。
  • setup() 或 tearDown()
    • 非 public 方法，其返回類型不是 void 或接受實參。
  • 由 @BeforeClass、@AfterClass、@BeforeAll 或 @AfterAll 註解的非 public，非 static 方法，其返回類型不是 void 或沒有有效的參數列表。
  • 由 @Before、@After、@BeforeEach 或 AfterEach 註解的非 public 方法，其返回類型不是 void 或接受實參。
  • 在 @BeforeAll 或 @AfterAll 方法中注入的 RepetitionInfo。
  • 由 @Test 註解測試使用的，在 @BeforeEach 或 @AfterEach方法中注入的 RepetitionInfo。
  • 由 @DataPoint 或 @DataPoints 註解的非 public 或非 static 欄位和方法。
  • 由 @Rule 註解的非 public 或非 TestRule 或 MethodRule 的子類型的欄位和方法。
  • 由 @ClassRule 註解的非 public，非 static 或非 TestRule 的子類型的欄位和方法。
  • TestCase 內帶有 test 前綴的非 public 方法，其返回類型不是 void，接受實參或為 static。
  • 由 @Test 註解的非 public 方法，其返回類型不是 void，接受實參或為 static。

格式錯誤的 @Before 方法範例 (Java)：

@Before private int foo(int arg) { ... }

在套用快速修復後：

@Before public void foo() { ... }

缺少方法原始碼範例 (Kotlin)：


  class Example {
    @MethodSource("parameters")
    @ParameterizedTest
    fun foo(param: String) { ... }
  }

在套用快速修復後：


  class Example {
    @MethodSource("parameters")
    @ParameterizedTest
    fun foo(param: String) { ... }

    companion object {
      @JvmStatic
      fun parameters(): Stream<Arguments> {
        TODO("Not yet implemented")
      }
    }
  }

使用檢查選項可以指定註解。 不會在測試方法中報告任何使用其中一種註解進行註解的參數。
