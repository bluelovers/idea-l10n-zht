報告註解類的構造函數的 @JvmOverloads，因為其毫無意義。

生成的字節碼和 Kotlin 元資料中沒有 @JvmOverloads 的佔用空間，因此 @JvmOverloads 不會影響生成的字節碼和程式碼行為。

自 Kotlin 1.4 起，註解類構造函數上的 @JvmOverloads 會導致編譯錯誤。

示例：


  annotation class A @JvmOverloads function Object() { [native code] }(val x: Int = 1)

在應用快速修復後：


  annotation class A function Object() { [native code] }(val x: Int = 1)
