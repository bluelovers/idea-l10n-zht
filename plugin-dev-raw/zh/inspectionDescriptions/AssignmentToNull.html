報告在宣告之外賦值給 null 的變數。

Java 中 null 的主要用途是表示未初始化的引用變數。 在極少數情況下，將變數顯式賦值給 null 有助於垃圾回收。 但是，使用 null 來表示缺失的、未指定的、無效的值或未找到的元素被認為是不良做法，可能會使您的程式碼更容易出現 NullPointerExceptions。 請改為考慮定義具有預期語意的 sentinel 物件或使用像 Optional 這樣的庫類型來表示沒有值。

範例:


  Integer convert(String s) {
    Integer value;
    try {
      value = Integer.parseInt(s);
    } catch (NumberFormatException e) {
      // 警告：null 用於表示“無效值”
      value = null;
    }
    return value;
  }

使用忽略對欄位的賦值選項可忽略對欄位的賦值。
