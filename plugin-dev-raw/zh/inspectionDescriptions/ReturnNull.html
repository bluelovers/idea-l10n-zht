報告返回值為 null 的 return 語句。 雖然這種結構偶爾有用，但可能導致程式碼因更容易出現 NullPointerException 而失敗。

如果方法設計為返回 null，建議使用 @Nullable 註解來標記 - 此類方法將被此檢查忽略。

範例:


  class Person {
    public String getName () {
      return null;
    }
  }

在套用快速修復後：


  class Person {
    @Nullable
    public String getName () {
      return null;
    }
  }

如果返回類型為 java.util.Optional，建議使用另外的快速修復將 null 轉換為 Otional.empty()。

使用以下選項設定檢查：

  • 是否忽略 private 方法。 這也將忽略從匿名類和 lambda 返回的 null。
  • 應報告陣列返回的 null 值、集合物件返回的值、普通物件返回的值還是三者的組合。 總是報告返回類型為 java.util.Optional 的方法中返回的 null。
  • 點擊設定註解以指定哪些註解應視為 'nullable'。
