報告未安全關閉的 I/O 資源。 此檢查所檢查的 I/O 資源包括 java.io.InputStream、java.io.OutputStream、java.io.Reader、java.io.Writer、java.util.zip.ZipFile、java.io.Closeable 和 java.io.RandomAccessFile。

不報告由其他 I/O 資源包裝的 I/O 資源，因為包裝資源將關閉被包裝資源。

預設情況下，該檢查假定資源可以由名稱中帶有 'close' 或 'cleanup' 的任何方法關閉。

範例：


  void save() throws IOException {
    FileWriter writer = new FileWriter("filename.txt"); //警告
    writer.write("sample");
  }

使用以下選項組態檢查：

  • 列出不需關閉並且此檢查應當忽略的 I/O 資源類別。
  • 是否允許在 try 塊內開啟 I/O 資源。 這種樣式不太合意，因為它比在 try 塊前面開啟資源更冗長。
  • 資源能否通過任何將資源作為實參傳遞的方法呼叫來關閉。
