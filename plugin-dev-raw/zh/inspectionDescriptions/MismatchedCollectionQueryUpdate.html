報告其內容已查詢但未更新或已更新但未查詢的集合。

此類別不一致查詢和更新毫無意義，可能表明有死碼或拼寫錯誤。

使用檢查設定可以指定與更新/查詢方法相對應的名稱模式。 返回元素的查詢方法會被自動檢測，並且只需指定將資料寫入輸出參數的查詢方法（例如 OutputStream）。

範例：

假設您有一個包含方法 store() 的自訂 FixedStack 類別： 


  public class FixedStack<T> extends Collection<T> {
    public T store(T t) {
      // 實作
    }
  }

您可以將 store 新增到更新方法表中，以報告不符合的查詢，例如： 


  void test(int i) {
    FixedStack<Integer> stack = new FixedStack<>();
    stack.store(i);
  }
