報告實作 java.io.Serializable 但不定義 serialVersionUID 欄位的非 static 內部類。

如果沒有 serialVersionUID 欄位，對該類做出的任何更改都將導致以前序列化的版本不可讀。 強烈建議 Serializable 非 static 內部類包含 serialVersionUID 欄位，否則由於合成存取器方法的差異，預設序列化演算法可能導致編譯器之間的序列化版本不相容。

建議通過快速修復添加缺失的 serialVersionUID 欄位。

範例：


  class Outer {
      class Inner implements Serializable {}
  }

在套用快速修復後：


  class Outer {
      class Inner implements Serializable {
          private static final long serialVersionUID = -7004458730436243902L;
      }
  }

使用以下選項設定檢查：

  • 列出此檢查不應報告其繼承者的類。 這適用於從超類繼承 Serializable 但不用於序列化的類。
  • 是否忽略 Serializable 匿名類。
