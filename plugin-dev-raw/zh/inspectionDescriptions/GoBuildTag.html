報告建置標記的格式錯誤和放置在錯誤位置的建置標記。 go 工具要求將建置標記放置在特定位置並以特殊語法編寫。 如果不滿足這些要求，則可能會忽略建置標記或可能會意外地從建置中刪除檔案。

請參閱 golang.org 上的 Build Constraints 頁面。

範例：

package main

// +build ignore

func main() {}

// +build ignore 部分應該在軟體套件宣告之前。 要解決此問題，您可以套用在軟體套件前面放置建置標籤快速修復。 在套用快速修復後：

// +build ignore

package main

import "fmt"
