報告對 toString() 或 substring() 等可取代為更簡單表達式的 String 構造函數和方法的冗餘調用。

例如，在 "string".substring(0)、"string".toString() 或 new StringBuilder().toString().substring(1,3) 等程式碼中，可以安全移除對這些方法的調用。

範例：


  System.out.println(new String("message"));

在套用快速修復後：


  System.out.println("message");

請注意，該快速修復會移除冗餘的構造函數調用，這可能會影響 String 引用相等。 即使被認為是一種不好的做法，如果您仍需要保留，請禁止警告或使用檢查設定來忽略冗餘的 String 構造函數調用。

請使用下方的不報告 String 構造函數調用選項來不報告程式碼（如上例所示）。 這將避免在使用 new String() 調用保證不同物件相等性的程式碼中套用快速修復後改變與 == 或 != 進行字串比較的結果。
