報告當 Runnable 傳遞給 Cleaner.register() 捕獲引用時會導致內存泄漏的情況。

此問題的可能來源：

  • 使用非 static 方法、字段或 this 本身的 lambda
  • 非 static 內部類（匿名與否）始終捕獲此引用
  • 實例方法引用
  • 從非 static 內部類訪問外部類非 static 成員

將報告的程式碼示例： 


  int fileDescriptor;
  Cleaner.Cleanable cleanable = Cleaner.create().register(this, () -> {
    System.out.println("adsad");
    //this 通過 fileDescriptor 捕獲
    fileDescriptor = 0;
  });

僅當專案或模塊的語言級別為 9 或更高時，此檢查才會報告。
