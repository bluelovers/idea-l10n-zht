報告始終產生相同結果、可能拋出異常或違反可 null 性約定的代碼結構。

示例：

if (array.length < index) {
  System.out.println(array[index]);
} // 數組索引始終超出界限

if (str == null) System.out.println("str is null");
System.out.println(str.trim());
// 最後一個語句可能會拋出 NPE

@NotNull
Integer square(@Nullable Integer input) {
    // 違反方法約定
    return input == null ? null : input * input;
}

檢查行為可能由許多注解控制，例如可 null 性注解、@Contract 注解、@Range 注解等。

配置檢查：

  • 使用對可 null 值的方法/字段/形參建議使用 @Nullable 注解選項可以在以下情況下發出警告：null 值作為實參傳遞給具有非注解形參的方法時，存儲到非注解字段時，或從非注解方法返回時。 在這種情況下，該檢查將建議傳播 @Nullable 注解。 您還可以使用配置注解按鈕配置可 null 性注解。
  • 使用將未注解的成員和形參視為 @Nullable 選項可假設非注解成員可以為 null，因此不得在非 null 上下文中使用。
  • 使用報告使用 null 文字實參用法的非 null 必需形參>選項可以報告不能為 null（例如在方法體中立即取消引用），但存在傳遞 null 文字的調用點的方法形參。
  • 使用報告始終返回非 null 值的可 null 方法選項，可報告注解為 @Nullable 但始終返回非 null 值的方法。 在這種情況下，建議您將注解更改為 @NotNull。
  • 使用不報告條件靜態證明始終為 true 的斷言選項可避免報告靜態證明始終為 true 的斷言。 這還包括 if (alwaysFalseCondition) throw new IllegalArgumentException(); 等條件。
  • 使用忽略 assert 語句選項可控制檢查如何處理 assert 語句。 預設情況下，該選項被禁用，這意味著斷言視為被執行（-ea 模式）。 如果啟用該選項，斷言將被完全忽略（-da 模式）。
  • 使用讀取保證為常量的值時發出警告選項可在讀取包含某些常量值（如true、false 或 null）的變量時添加警告。
  • 使用報告僅在某些代碼路徑上發生的問題選項可控制是否報告僅在某些代碼路徑上發生的問題。 如果禁用此選項，則不會報告 exception is possible 等警告。 該檢查只會報告類似 exception will definitely occur 的警告。 這種模式可以大大減少誤報的數量，特別是如果代碼沒有一致地使用可 null 性和約定注解進行注解。 這就是為什麼它對於查找傳統代碼庫中最重要的問題很有用。
