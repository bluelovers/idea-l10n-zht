報告與超類的不可存取方法包含相同簽名的方法， 例如，另一個軟體套件中的超類的 private 方法或 package-private 方法。

此類方法名稱可能令人困惑，因為子類中的方法看似覆寫，實則隱藏了超類不可存取的方法。 此外，如果此方法在超類中的可見性日後發生變化，可能會靜默地更改子類的語意，否則會導致編譯錯誤。

建議通過快速修復重命名該方法。

範例：


  public class Super {
    private void test() {
    }
  }

  public class Sub extends Super {
    void test() { // 將 'Super.test()' 設為 public 將導致編譯錯誤
                  // 將 'Super.test()' 設為 package-private 會使 'Sub.test()' 變成覆寫
    }
  }
