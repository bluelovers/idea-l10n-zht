報告任何從 long、integer、short 或 byte 基元參數實例化新的 Long、Integer、Short 或 Byte 物件的嘗試。

建議使用 Java 5 中引入的靜態方法 valueOf() 預設情況下，此方法快取值介於 -128 到 127 (包含在內)之間的物件。

範例:


  Integer i = new Integer(1);
  Long l = new Long(1L);

套用快速修復後，程式碼變為:


  Integer i = Integer.valueOf(1);
  Long l = Long.valueOf(1L);

僅當專案或模組的語言級別為 5 級或更高級別時，此檢查才進行報告。

使用忽略具有字串參數的新數字表達式選項，可以忽略對包含 String 參數的數字構造函數的調用。

使用僅在構造函數為 @Deprecated 時報告選項，僅報告對棄用的構造函數的調用。 自 JDK 9 起已棄用 Long、Integer、Short 和 Byte 構造函數。
