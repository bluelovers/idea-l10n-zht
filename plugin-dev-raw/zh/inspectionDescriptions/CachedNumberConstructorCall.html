報告具有基元 long、integer、short 或 byte 實參的新 Long、Integer、Short 或 Byte 物件的實例化。

建議使用 Java 5 中引入的 static 方法 valueOf() 預設情況下，此方法快取值介於 -128 到 127（包含）之間的物件。

範例：


  Integer i = new Integer(1);
  Long l = new Long(1L);

套用快速修復後，程式碼變為：


  Integer i = Integer.valueOf(1);
  Long l = Long.valueOf(1L);

僅當專案或模組的語言級別為 5 或更高時，此檢查才會報告。

使用忽略具有字串實參的新數字表達式選項可以忽略對包含 String 實參的數字建構函式的呼叫。

使用僅在建構函式為 @Deprecated 時報告選項，僅報告對棄用的建構函式的呼叫。 Long、Integer、Short 和 Byte 建構函式自 JDK 9 起已被棄用。
