報告被認為太長的關鍵字。 不同資料庫的長度限制不同。

範例（MySQL）：

CREATE TABLE ooooooooooooooooooooooooooooooooooooooooooooooooooooooooooooooooo
(
   ooooooooooooooooooooooooooooooooooooooooooooooooooooooooooooooooo int
);

在 MySQL 中，關鍵字不得超過 64 個字符。

請考慮以下不同資料庫的限制：

  • Sybase ASE：

    • 帶引號的名稱：253 個字符
    • 未加引號的名稱：255 個字符

  • Db2 LUW：

    • BUFFER POOL、INDEX EXTENSION、TABLESPACE、TRANSFORM、TYPE MAPPING：18 個字符
    • 其他物件：128 個字符
  • Db2 iSeries：128 個字符

  • Db2 z/OS：

    • DATABASE 和 TABLESPACE：8 個字符
    • 列和帶有 WITH RETURN 的游標：30 字符
    • 其他物件：128 個字符
  • Derby：128 個字符
  • Exasol：128 個字符
  • HSQLDB：128 個字符

  • MySQL：

    • 查詢中的別名： 256 個字符
    • USER 和 ROLE：32 個字符
    • 其他物件：64 個字符

  • MariaDB：

    • USER：80 個字符
    • ROLE： 128 個字符
    • 其他物件：與 MySQL 相同
  • Microsoft SQL Server：128 個字符
  • Vertica：128 個字符
  • SQL 92：128 個字符
  • Snowflake：255 個字符
  • Redshift：127 個字符
  • PostgreSQL/Greenplum/Cockroach：63 個字符

  • Oracle：

    • 長名稱被啟用（database.oracle.names.long 被指定，Oracle 版本早於 12.2或未知）：

      • DISKGROUP、ROLLBACK SEGMENT、TABLESPACE、PLUGGABLE DATABASE：30 個字符
      • DATABASE：8 個字符
      • 其他物件：128 個字符
    • 長名稱被停用：
      • DATABASE：8 個字符
      • DATABASE LINK：128 個字符
      • 其他物件：30 個字符
