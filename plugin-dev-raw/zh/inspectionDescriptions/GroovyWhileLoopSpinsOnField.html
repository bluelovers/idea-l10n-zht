報告在非 volatile 欄位值上自旋，等待它被另一個執行緒更改的 while 迴圈。

除了在迴圈內完成少量工作時可能會非常耗費 CPU 之外，此類別迴圈很可能具有與預期不同的語意。 即便另一個執行緒改變了該欄位的值，Java 記憶體模型也允許該迴圈永不完成。

範例：


  class SpinsOnField {
      boolean ready = false;

      void run() {
          // 即使在從其他執行緒呼叫 markAsReady 後，
          // 該迴圈也可能永不完成
          while (!ready) {
          }
          // 執行某些工作
      }

      void markAsReady() {
          ready = true;
      }
  }

此外，自 Java 9 起，建議在 volatile 欄位的自旋迴圈中呼叫 Thread.onSpinWait()，這可能會顯著提高在某些硬體上的性能。
