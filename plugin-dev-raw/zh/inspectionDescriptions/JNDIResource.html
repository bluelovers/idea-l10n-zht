報告未安全關閉的 JNDI 資源。 此檢查報告的 JNDI 資源包括 javax.naming.InitialContext和 javax.naming.NamingEnumeration。

預設情況下，該檢查假定資源可以由名稱中帶有 'close' 或 'cleanup' 的任何方法關閉。

範例：


  Object findObject(Properties properties, String name) throws NamingException {
    Context context = new InitialContext(properties); //上下文未關閉
    return context.lookup(name);
  }

使用以下選項組態檢查：

  • 是否允許在 try 塊內開啟 JNDI 資源。 這種樣式不太合意，因為它比在 try 塊前面開啟資源更冗長。
  • 資源能否通過任何將資源作為實參傳遞的方法呼叫來關閉。
