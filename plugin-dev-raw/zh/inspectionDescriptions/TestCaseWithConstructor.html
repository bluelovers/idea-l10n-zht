報告 JUnit 測試用例及其構造函數中的初始化邏輯。 如果構造函數失敗，將不會調用 @After 注解方法或 tearDown() 方法。 這可能導致測試環境部分初始化，因而對其他測試產生不利影響。 相反，JUnit 測試用例初始化應在 setUp() 或 @Before 注解方法中完成。

不良示例：


  public class ImportantTest {
    private File file;
    public ImportantTest() throws IOException {
      file = File.createTempFile("xyz", ".tmp");
    }

    // ... 測試轉到此處
  }
