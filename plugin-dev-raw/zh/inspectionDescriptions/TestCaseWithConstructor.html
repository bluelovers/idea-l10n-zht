報告在其構造函數中具有初始化邏輯的測試用例。 如果構造函數失敗，將不會調用 @After 註解方法或 tearDown() 方法。 這可能導致測試環境部分初始化，因而對其他測試產生不利影響。 相反，測試用例初始化應在 setUp() 或 @Before 註解方法中完成。

不良範例：


  public class ImportantTest {
    private File file;

    public ImportantTest() throws IOException {
      file = File.createTempFile("xyz", ".tmp");
    }

    // ... tests go here
  }
