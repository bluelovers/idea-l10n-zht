報告在依賴於 kotlinx.coroutines 庫的程式碼中的 kotlin.coroutine.coroutineContext 屬性的用法。

當代碼中同時存在 kotlin.coroutine.coroutineContext 和 kotlinx.coroutines.CoroutineScope.coroutineContext 時，可能導致混淆和潛在 bug。

kotlinx.coroutines 庫提供了 currentCoroutineContext() 函式作為更清晰的替代方案，即使沒有明顯的衝突，也應優先選擇。

如需了解詳情，請參閱 kotlin.coroutine.coroutineContext 和 kotlinx.coroutines.currentCoroutineContext 的文檔。

範例：


  suspend fun getCurrentJob(): Job? {
    return coroutineContext[Job]
  }

在套用快速修復後：


  suspend fun getCurrentJob(): Job? {
    return currentCoroutineContext()[Job]
  }
