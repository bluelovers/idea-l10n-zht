報告直接呼叫 panic() 或 recover() 的 defer 和 go 語句。

此類別語句很少有用，可能意味著誤用了 panic() 和 recover() 機制。 應特別注意以下事項：

  • go panic()：一個新啟動的 goroutine 會立即 panic。
  • defer panic()：帶有這個語句的函式在退出時總是會 panic。
  • go recovery()：沒有效果，因為新啟動的 goroutine 不能 panic。
  • defer recovery()：帶有這個語句的函式將默默地停止 panic。 這種用法可能有效，但一種慣用的方法是檢查 recover() 返回的值： 
    defer func() {
  if r := recover(); r != nil {
    fmt.Println("Recovered from: ", r)
  }
}()

有關 go 語句和 panic 處理的更多資訊，請參閱 Go 語言規範中的 Handling panics 和 Go statements。
