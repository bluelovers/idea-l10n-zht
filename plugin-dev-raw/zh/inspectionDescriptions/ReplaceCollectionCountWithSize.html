報告對 Collection<T>.count() 的呼叫。

此函式呼叫可以取代為 .size。

.size 形式可以確保運算是 O(1) 並且不會分配額外的物件，而 count() 可能與 Iterable<T>.count() 混淆，後者是 O(n) 和分配。

範例：

  fun foo() {
      var list = listOf(1,2,3)
      list.count() // 可取代的 'count()'
  }

在套用快速修復後：

  fun foo() {
      var list = listOf(1,2,3)
      list.size
  }
