報告 equals()、hashCode()、compareTo() 和 Comparator.compare() 方法內（臨時）新物件的建構。

除了建構函式呼叫之外，還可以通過在 foreach 語句中自動裝箱或迭代器建立來建立新物件。 這可能會導致性能問題，例如，將物件新增到 Set 或 Map 時，這些方法將在這些地方被經常呼叫。

當在 throw 或 assert 語句中建立物件時，該檢查將不會報告。

範例：


  class Person {
    private String name;
    private int age;

    public boolean equals(Object o) {
      return Arrays.equals(new Object[] {name, age}, new Object[] {((Foo)o).name, ((Foo)o).age});
    }

    public int hashCode() {
      return (name + age).hashCode();
    }
  }

在此範例中，在 equals() 內部建立了兩個額外的陣列，age 欄位的用法需要裝箱，而 name + age 隱式地建立了一個新字串。
