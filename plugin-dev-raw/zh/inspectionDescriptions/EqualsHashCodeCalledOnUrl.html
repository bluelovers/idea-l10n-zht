報告在 java.net.URL 物件上調用的 hashCode() 和 equals()。

java.net.URL 類會在內部使用 java.net.URLStreamHandler 的實例來執行 hashCode() 和 equals() 。 這會導致性能問題，因為 java.net.URLStreamHandler 會執行 DNS 尋找。 根據網絡的可用性和 DNS 伺服器的速度，這可能會導致顯著延遲。

該問題很可能可以通過改為使用 java.net.URI 加以解決。

範例：


  int f(URL url1, URL url2) {
    if (url1.equals(url2)) return url1.hashCode();
    else return url2.hashCode();
  }
