報告可以轉換為記錄類別的類別。

記錄類別側重於對不可變數據進行建模，而不是對可擴展行為進行建模。 資料驅動方法的自動隱式實作（例如 equals() 和存取器）有助於減少樣板程式碼。

請注意，不是每個類別都可以是記錄類別。 以下是一些限制：

  • 該類不能有子類別。
  • 類別中的所有非 static 欄位都必須為 final。
  • 不得存在初始設定式、泛型建構函式和原生方法。

有關記錄類別的完整描述，請參閱 Java 語言規範。

範例：


  class Point {
    private final double x;
    private final double y;

    Point(double x, double y) {
      this.x = x;
      this.y = y;
    }

    double getX() {
      return x;
    }

    double getY() {
      return y;
    }
  }

在套用快速修復後：


  record Point(int x, int y) {
  }

啟用建議重新命名存取器方法選項可以自動將 getX()/isX() 存取器重命名為 x()。

使用如果成員變得更易於存取，則停用醒目提示選項以排除那些成員的可存取性因轉換而變弱的類別。 快速修復將始終作為一個意圖可用，而且觸發它將顯示受影響的成員並要求確認。 在批處理模式下，將不建議進行轉換。

使用禁止轉換有以下註解的類別列表可在要轉換的類別中排除其註解符合指定模式的類別。
