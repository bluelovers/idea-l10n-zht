建議用記錄取代類。

當您需要專注於建模不可變資料而不是可擴展行為時，檢查會很有用。 資料驅動方法的自動實作，例如 equals 和存取器，有助於消除樣板。

請注意，不是每個類都可以成為記錄。 以下是一些限制：

  • 類不能包含繼承者，並且必須是一個頂級類。
  • 類中的所有非 static 欄位必須為 final。
  • 不允許類包含實例初始值設定項、泛型構造函數或原生方法。

要獲取完整的限制列表，請參閱 Oracle 文檔。

範例：


  class Point {
    private final double x;
    private final double y;

    Point(double x, double y) {
      this.x = x;
      this.y = y;
    }

    double getX() {
      return x;
    }

    double getY() {
      return y;
    }
  }

此記錄將轉換為：


  record Point(int x, int y) {
  }

僅當專案或模組的語言級別為 16 或更高時，此檢查才會報告。
