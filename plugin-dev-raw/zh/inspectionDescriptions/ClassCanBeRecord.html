建議用記錄取代類別。

當您需要專注於建模不可變資料而不是可擴展行為時，檢查會很有用。 資料驅動方法的自動實作，例如 equals 和存取器，有助於消除樣板。

請注意，不是每個類別都可以成為記錄。 以下是一些限制：

  • 類別不能包含繼承者，並且必須是一個頂級類別。
  • 類別中的所有非 static 欄位必須為 final。
  • 不允許類別包含實例初始設定式、泛型建構函式或原生方法。

要獲取完整的限制列表，請參閱 Oracle 文檔。

範例：


  class Point {
    private final double x;
    private final double y;

    Point(double x, double y) {
      this.x = x;
      this.y = y;
    }

    double getX() {
      return x;
    }

    double getY() {
      return y;
    }
  }

在套用快速修復後：


  record Point(int x, int y) {
  }

啟用建議重新命名 get/is 存取器選項以允許將 getX()/isX() 存取器自動重命名為 x()。

使用當轉換使成員更易於存取時選項可指定轉換是否可以違反類別封裝：

  • 選擇不建議轉換選項則從不違反類別封裝
  • 選擇在衝突檢視中顯示受影響的成員選項則套用轉換並通知有關封裝違規的問題
  • 選擇無提示轉換選項則無論是否存在封裝違規問題，均可無提示套用轉換

使用禁止轉換有以下註解的類別列表可在要轉換的類別中排除其註解符合指定模式的類別。

僅當專案或模組的語言級別為 16 或更高時，此檢查才會報告。
