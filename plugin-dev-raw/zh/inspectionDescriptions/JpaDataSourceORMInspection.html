如果無法在組態的資料來源中解析，則報告以下 Persistence ORM 註解中的參照：
  • Table
  • Column
  • [Any]To[Any]
  • JoinColumn / JoinTable
  • NamedQuery / NativeQuery
  • Embeddable

建議重新整理資料來源或分配不同的資料來源。

範例資料庫表：


  CREATE TABLE DATA_TABLE (
    ID INTEGER,
    DATA VARCHAR(100)
);

範例 JPA 實體類別：


  @Entity
  @Table(name = "UNKNOWN_TABLE") // 未解決的參照
  public class JavaEntity {
    @Column(name = "ID") // 無法解析未定義的表中的列
    Integer id;
  }

注意：資料來源可以通過 `分配資料來源` 動作或實體類別的相應裝訂區域圖標進行組態
