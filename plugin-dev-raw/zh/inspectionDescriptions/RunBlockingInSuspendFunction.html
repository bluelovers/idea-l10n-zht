報告 suspend 函式內部的 runBlocking 呼叫。

在 suspend 函式中使用 runBlocking 會阻塞呼叫執行緒，違背異步編程的目的。

該快速修復可以將 runBlocking 呼叫取代為以下備選項之一，具體取決於上下文：

  • run 呼叫。
  • 使用特定 CoroutineContext 時的 withContext 呼叫。
  • 無 runBlocking 包裝器的直接內聯程式碼。

範例：


suspend fun something() {
  runBlocking {
    code() // 執行緒在此處被阻塞
  }
}

在套用快速修復後：


suspend fun something() {
  code() // 異步執行
}
