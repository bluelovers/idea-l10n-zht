報告可能在 Reactive Stream 鏈中返回 null 的轉換操作。

Reactive Streams 不支持 null 值，因為這些值會導致此類程式碼失敗。

範例:

repository.findWithTailableCursorBy()
    .map(e -> (Person)null)
    .doOnNext(System.out::println)

最好不要在 Reactive Stream 鏈程式碼內使用可 null 值，但如果需要如此，請使用特殊運算，如 Reactor 的 mapNotNull。

有一個快速修復可用於將 map 取代為 mapNotNull。 它會忽略異常，因為 mapNotNull 允許 null 值。

快速修復之前:

repository.findWithTailableCursorBy()
    .map(e -> (Person)null)
    .doOnNext(System.out::println)

在套用快速修復後：

repository.findWithTailableCursorBy()
    .mapNotNull(e -> (Person)null)
    .doOnNext(System.out::println)
