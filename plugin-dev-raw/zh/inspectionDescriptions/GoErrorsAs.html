當第二個實參不是一個指向接口或實作錯誤的類型的指針時，報告對 errors.As 函數的調用。

此類調用在執行時會出現 panic。

有關 As 函數的更多資訊，請參見 golang.org 上的 func As 頁面。

範例：

_, err := os.Open("non-existing")
var pathError *fs.PathError
if errors.As(err, pathError) { // 需要一個指向 *fs.PathError 的指針
}

在套用在前面追加 '&'快速修復後：

_, err := os.Open("non-existing")
var pathError *fs.PathError
if errors.As(err, &pathError) {
}

僅當語言版本為 1.13 或更高時，此檢查才會報告。
