報告條件被否定的條件表達式。

翻轉條件表達式分支的順序通常會增加此類別語句的清晰度。

使用忽略 '!= null' 比較和忽略 '!= 0' 比較選項可以忽略 obj != null 或 num != 0 形式的比較。 由於 obj != null 實際上意味著 "obj exists"，整個表達式的含義不涉及任何否定，所以易於理解。

同樣的推理也適用於 num !=0 表達式，在使用位元遮罩時更是如此。

這些形式還有另外的好處，即先提及有趣的情況。 在大多數情況下，== null 分支的值為 null 本身，如以下範例所示： 


    static String getName(Person p) {
        return p != null ? p.getName() : null;
    }

    static String getExecutableString(int fileMode) {
        return (fileMode & 0b001001001) != 0 ? "executable" : "non-executable";
    }
